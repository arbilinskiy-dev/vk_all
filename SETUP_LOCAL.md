# 🚀 Инструкция по локальному развертыванию проекта

Эта инструкция поможет развернуть проект **VK Content Planner** на вашем компьютере с локальной базой данных SQLite.

---

## 📋 Требования

- **Node.js** 18+ и npm (для фронтенда)
- **Python** 3.9+ (рекомендуется 3.11)
- **Git** (для клонирования репозитория)
- **VS Code** (опционально, но рекомендуется)

---

## ⚙️ Пошаговая настройка

### 1️⃣ Клонирование репозитория

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd <ПАПКА_ПРОЕКТА>
```

---

### 2️⃣ Настройка фронтенда

#### 2.1. Установка зависимостей

В **корневой папке проекта** выполните:

```bash
npm install
```

#### 2.2. Запуск фронтенда

```bash
npm run dev -- --host 0.0.0.0 --port 5173
```

Фронтенд будет доступен по адресу: **http://localhost:5173** (а также по IP вашей машины в локальной сети)

> ⚠️ **Важно:** Пока бэкенд не запущен, фронтенд не сможет загружать данные.

---

### 3️⃣ Настройка бэкенда (Python)

#### 3.1. Решение проблемы с Pillow

Если у вас возникла ошибка при установке `Pillow`, выполните следующие команды:

```bash
# Обновите инструменты установки
python -m pip install --upgrade pip setuptools wheel

# Очистите кэш
pip cache purge
```

#### 3.2. Создание виртуального окружения

Перейдите в папку `backend_python`:

```bash
cd backend_python
```

Создайте и активируйте виртуальное окружение:

**Windows (PowerShell):**
```bash
python -m venv venv
.\venv\Scripts\activate
```

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

#### 3.3. Установка зависимостей

```bash
pip install -r requirements.txt
```

#### 3.4. Создание файла `.env`

Создайте файл `.env` в папке `backend_python` со следующим содержимым:

```env
# Обязательные параметры
VK_USER_TOKEN="ваш_токен_вконтакте"
GEMINI_API_KEY="ваш_ключ_gemini_api"

# Учетные данные администратора (можно оставить по умолчанию)
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin

# Для локальной разработки DATABASE_URL НЕ УКАЗЫВАЙТЕ!
# Если эта переменная НЕ задана, автоматически будет использоваться SQLite
```

#### 3.5. Получение необходимых токенов

##### 🔹 VK_USER_TOKEN

1. Перейдите на сайт: https://vkhost.github.io/
2. Выберите **"VK Admin"**
3. Отметьте галочки:
   - ✅ Фотографии
   - ✅ Группы
   - ✅ Сообщения сообщества
   - ✅ Управление сообществом
4. Нажмите **"Разрешить"**
5. Скопируйте полученный токен в `.env`

##### 🔹 GEMINI_API_KEY

1. Перейдите на: https://aistudio.google.com/apikey
2. Создайте новый API-ключ
3. Скопируйте его в `.env`

#### 3.6. Запуск бэкенда

В папке `backend_python` с активированным виртуальным окружением выполните:

##### Режим разработки (1 воркер, hot-reload)

```bash
uvicorn main:app --reload
```

Бэкенд будет доступен по адресу: **http://127.0.0.1:8000**

##### Режим с несколькими воркерами (рекомендуется)

Для запуска с 2+ API-воркерами и отдельным Callback Worker:

```bash
python start_local.py
```

Это запустит:
- **2 uvicorn API-воркера** на порту 8000 (параллельная обработка запросов)
- **1 выделенный VK Callback Worker** (обработка событий из Redis-очереди)

Можно настроить через переменные окружения:
| Переменная | По умолчанию | Описание |
|---|---|---|
| `API_WORKERS` | `2` | Количество API-воркеров |
| `API_PORT` | `8000` | Порт API-сервера |
| `API_HOST` | `127.0.0.1` | Хост API-сервера |

Пример с 3 воркерами:
```bash
set API_WORKERS=3 && python start_local.py
```

> ⚠️ Режим `start_local.py` не поддерживает hot-reload. Для разработки с автоперезагрузкой используйте `uvicorn main:app --reload`.

При первом запуске автоматически создастся файл локальной базы данных **`vk_planner.db`** в папке `backend_python`.

## 🔗 Настройка SSH-туннеля (для VK Callback API)

Если вам нужно получать Callback-события от VK при локальной разработке, потребуется SSH reverse tunnel через VM.

**Схема работы:**
```
VK → https://api.dosmmit.ru/api/vk/callback-local
   → nginx (VM) → localhost:8001 (SSH tunnel)
   → ваш localhost:8000 (бэкенд)
```

**Требования:**
1. Бэкенд запущен на `localhost:8000`
2. SSH-ключ: `%USERPROFILE%\.ssh\id_ed25519`
3. VM доступна: `yc-user@93.77.184.105`

**Запуск туннеля:**

Вариант 1 — через готовый скрипт:
```bash
cd backend_python
start_tunnel.bat
```

Вариант 2 — вручную:
```bash
ssh -i "%USERPROFILE%\.ssh\id_ed25519" -o StrictHostKeyChecking=no -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -o ExitOnForwardFailure=yes -N -R 8001:localhost:8000 yc-user@93.77.184.105
```

После запуска туннеля используйте кнопку **«SSH Tunnel»** в настройках Callback на фронтенде.

> ⚠️ Для остановки туннеля нажмите `Ctrl+C` в терминале.

## ❗ Частые проблемы и решения

### Проблема: Pillow не устанавливается

**Решение:**
```bash
# Попробуйте установить более новую версию
pip install pillow==10.4.0

# Или установите отдельно с форсированной переустановкой
pip install --upgrade --force-reinstall pillow
```

### Проблема: Фронтенд не подключается к бэкенду

**Проверьте:**
1. Бэкенд запущен и доступен по адресу `http://127.0.0.1:8000`
2. На странице входа выбран режим **"Локальный"**
3. В консоли браузера (F12) нет ошибок CORS

### Проблема: Ошибка "ModuleNotFoundError"

**Убедитесь:**
1. Виртуальное окружение активировано (в терминале должно быть `(venv)` перед строкой)
2. Все зависимости установлены: `pip install -r requirements.txt`

### Проблема: База данных не создается

**Проверьте:**
1. В файле `.env` **НЕТ** строки `DATABASE_URL` (для SQLite она не нужна)
2. У вас есть права на запись в папку `backend_python`

## 🔄 Полезные команды

### Остановка серверов

- **Фронтенд/Бэкенд:** Нажмите `Ctrl+C` в терминале

### Перезапуск бэкенда

```bash
# В папке backend_python с активированным venv
uvicorn main:app --reload
```

### Просмотр логов

Все логи бэкенда выводятся в терминал, где запущен `uvicorn`.

### Сброс базы данных

Если нужно начать с чистой базы:

1. Остановите бэкенд (`Ctrl+C`)
2. Удалите файл `backend_python/vk_planner.db`
3. Запустите бэкенд снова — база пересоздастся автоматически
