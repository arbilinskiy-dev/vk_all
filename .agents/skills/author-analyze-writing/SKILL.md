# Скилл: Анализ стиля письма и речи

## Роль

Ты — лингвист-стилист и филолог с экспертизой в анализе идиолекта (индивидуального стиля речи). Твоя задача — разложить стиль автора на атомарные элементы, которые можно воспроизвести.

## Входные данные

Файл: `backend_python/data/author_texts_for_analysis.json`

## Процедура анализа

### 1. Лексический анализ

- **Словарный запас**: богатый/бедный, специализированный/общий
- **Любимые слова и выражения**: слова-паразиты, фирменные фразы (с примерами из текстов, указать частоту)
- **Сленг и жаргон**: какой используется и когда
- **Заимствования**: англицизмы, профессионализмы, IT-термины
- **Разговорность vs книжность**: соотношение, в каких контекстах что
- **Уменьшительные/ласкательные**: использует ли, как часто
- **Междометия и восклицания**: какие характерны
- **Мат/грубость**: есть ли, в каком контексте, как маскирует
- **Фирменные неологизмы**: придумывает ли свои слова/конструкции

### 2. Синтаксический анализ

- **Длина предложений**: средняя, вариативность (короткие рубленые vs длинные потоковые)
- **Тип предложений**: простые/сложные, сочинение/подчинение
- **Инверсии**: переставляет ли слова для эффекта
- **Парцелляция**: дробит ли предложения на короткие куски
- **Вводные конструкции**: какие типичны
- **Обращения к читателю**: прямые обращения, вопросы
- **Незаконченные предложения**: использует ли троточия, обрывы мысли
- **Диалогизация**: вставляет ли воображаемый диалог или внутренний монолог

### 3. Пунктуация и форматирование текста

- **Эмодзи**: какие и как часто, есть ли фирменные
- **Знаки препинания**: злоупотребление скобками, тире, многоточиями, восклицательными
- **Абзацирование**: длина абзацев, есть ли паттерн
- **Выделение**: капс, курсив, жирный — как использует
- **Нумерация и списки**: использует ли
- **Пустые строки**: как разделяет блоки текста

### 4. Морфологические особенности

- **Время**: настоящее/прошедшее/будущее — какое преобладает
- **Лицо**: первое/второе/третье — как переключается
- **Наклонение**: повелительное, сослагательное — когда
- **Степени сравнения**: использует ли превосходную степень, гиперболы
- **Частицы и союзы**: фирменные (ну, вот, типа, короче, собственно)

### 5. Регистры и код-свитчинг

- **Переключение регистров**: когда официальный, когда разговорный, когда грубый
- **Контекстность**: как тема влияет на стиль
- **Аудитория**: адаптирует ли стиль под разные темы
- **Эволюция стиля**: как менялся стиль по годам

### 6. Фирменные конструкции (КРИТИЧНО для воспроизведения)

Извлечь **конкретные шаблоны** с примерами:
- Как начинает пост (первые 1-3 слова/фразы — собрать ВСЕ варианты)
- Как заканчивает пост (последнее предложение — паттерны)
- Как делает переходы между мыслями
- Как вводит тему поста
- Как делает выводы/резюме
- Как задаёт вопросы читателю
- Как цитирует/ссылается
- Характерные связки между абзацами

### 7. Количественные метрики

- Средняя длина поста (слов)
- Медианная длина поста
- Распределение по длине (короткие <50, средние 50-200, длинные >200)
- Среднее число абзацев на пост
- TTR (Type-Token Ratio) — лексическое разнообразие
- Топ-50 самых частых слов (без стоп-слов)
- Топ-20 биграмм

## Формат результата

```json
{
  "skill": "author-analyze-writing",
  "analysis": {
    "lexical": { ... },
    "syntactic": { ... },
    "punctuation_formatting": { ... },
    "morphology": { ... },
    "registers": { ... },
    "signature_patterns": {
      "post_openings": ["паттерн 1 (пример)", ...],
      "post_closings": ["паттерн 1 (пример)", ...],
      "transitions": ["паттерн 1 (пример)", ...],
      "conclusions": ["паттерн 1 (пример)", ...]
    },
    "quantitative": { ... },
    "for_author_skill": "компактная инструкция 500-800 слов: точные правила воспроизведения стиля — что делать, что не делать, с конкретными примерами конструкций"
  }
}
```

**Критично:** `for_author_skill` — это набор КОНКРЕТНЫХ ПРАВИЛ для копирования стиля. Не абстрактные характеристики, а actionable инструкции: «Начинай пост с...», «Используй слова...», «Длина предложения обычно...», «Эмодзи ставь...».

## Делегирование саб-агентам

Атомарная задача для саб-агента:
1. Прочитать `author_texts_for_analysis.json`
2. Провести лингвистический анализ
3. Сохранить в `backend_python/data/author_analysis/writing_style.json`
4. Вернуть `for_author_skill`
