# Documentation Workflow — Форматы записей в docs/sandbox/

## Формат записи выявленной особенности VK API

Записывается в `docs/sandbox/vk_api_findings.md` (или `google_ai_findings.md` для Google AI).

```markdown
### 🔴/🟡/🟢 Краткое название

**Проблема:** Что пошло не так (поведение API).

**Причина:** Почему API ведёт себя так (техническая причина).

**Решение:** Что сделали в коде для обхода/исправления.

**Пример кода:**
```python
# Исправленный вариант
```
```

Маркировка серьёзности:
- 🔴 **Критическое** — блокирует работу, обязательно обходить
- 🟡 **Важное** — влияет на логику, нужно учитывать
- 🟢 **Информационное** — полезно знать, не критично

## Формат записи итерации отладки

Записывается в `docs/sandbox/debug_chronology.md`.

```markdown
### Итерация N: Краткое название

- **Цель:** Что пытались сделать.
- **Цепочка:** `метод1` [токен] → `метод2` [токен] → ...
- **Результат:** ✅ / ❌ / ⏳ + описание.
- **Причина (если ❌):** Техническая причина провала.
- **Решение (если ✅):** Что именно поменяли.
```

---

## Формат описания нового теста

Создаётся как `docs/sandbox/testN_название.md`.

```markdown
# Тест N: Название

## Цель
Что тестируем и зачем.

## Цепочка вызовов
`метод1` [user_token] → `метод2` [group_token] → ...

## Эндпоинты бэкенда
- `POST /sandbox/test-name/execute` — основной запуск
- `GET /sandbox/test-name/status` — статус (если есть)

## Параметры
| Параметр | Тип | Описание |
|---|---|---|
| group_id | int | ID группы VK |

## Результаты
### Итерация 1: Название
- **Цель:** ...
- **Результат:** ✅ / ❌
```

---

## Обновление оглавления

При создании нового теста добавить строку в таблицу `docs/sandbox/README.md`:

```markdown
| testN | Название теста | ⏳ В работе | Итерация 1: Начало |
```

---

## Полный список файлов для чтения при записи итогов

1. `docs/sandbox/README.md` — оглавление + статусы
2. `docs/sandbox/testN_*.md` — файл конкретного теста
3. `docs/sandbox/vk_api_findings.md` — особенности VK API
4. `docs/sandbox/google_ai_findings.md` — особенности Google AI (если применимо)
5. `docs/sandbox/debug_chronology.md` — хронология отладки
6. `backend_python/services/sandbox/*.py` — код сервиса теста
7. `backend_python/routers/sandbox.py` — роутер
8. `features/sandbox/` — фронтенд-хуки и компоненты
9. Контекст диалога — что тестировали, баги, решения
