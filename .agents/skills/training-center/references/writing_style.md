# Writing Style — Стиль написания обучающих материалов

## Контекст

**«Планировщик контента»** — внутренний инструмент SMM-агентства для собственных сотрудников. НЕ коммерческий продукт.

## Целевая аудитория

### 1. SMM-специалисты (основные)
- Работа с контентом: тексты, посты, конкурсы, товары, модерация предложки
- Боль без приложения: ручное копирование постов в 10 сообществ, нет общей картины
- С приложением: всё в одном месте, AI-редактор, мультипроектная публикация

### 2. Технические специалисты
- Настройка интеграций, товары, подключение сервисов
- С приложением: автоматизация историй, управление настройками

### 3. Руководители
- Контроль специалистов, работа с заказчиками
- С приложением: цветовые счётчики, история действий, обзор проектов

## Запрещённый стиль

- Фантазии про «фрилансеров», «ваш бизнес», «если вы блогер»
- Маркетинговый лендинг
- Абстрактные сценарии
- Технический жаргон (hover, tooltip, callback, state и т.д.)

## Обязательный стиль

- Как внутренняя документация для коллег
- Конкретные примеры из работы агентства
- «Было/стало» — как делали раньше vs как делаем теперь
- Фокус на экономии времени
- Понятный язык для обычных пользователей

## Структура текста (формат «Вопрос-Ответ»)

- **«Что это такое?»** — краткое описание функции
- **«Как с этим работать?»** — пошаговые инструкции
- **«Почему это так?»** — объяснение логики
- **«Что можно?» / «Что нельзя?»** — возможности и ограничения

## Замена технических терминов

| Технический термин | Понятная замена |
|---|---|
| Hover / Ховер-эффект | «Наведите курсор», «При наведении» |
| Tooltip / Тултип | «Подсказка», «Всплывающая подсказка» |
| Callback | «После выполнения», «Когда завершится» |
| State / Стейт | «Состояние», «Текущий режим» |
| Render / Рендер | «Отображение», «Показ» |
| Props | «Настройки», «Параметры» |
| Modal / Модалка | «Всплывающее окно», «Диалоговое окно» |
| Dropdown | «Выпадающий список», «Список выбора» |
| Toggle | «Переключатель», «Включить/выключить» |
| Loader / Spinner | «Индикатор загрузки», «Загрузка...» |
| Placeholder | «Подсказка в поле», «Пример текста» |
| Disabled | «Недоступно», «Заблокировано» |
| Validation | «Проверка», «Проверка правильности» |

## Примеры тона

❌ «Если вы фрилансер и ведёте несколько клиентов...»
✅ «Когда нужно опубликовать один пост в 10 проектах, раньше приходилось заходить в каждое сообщество. Теперь — одна кнопка.»

❌ «Этот инструмент поможет вашему бизнесу расти...»
✅ «Счётчик показывает 0 — значит в проекте закончились посты, пора планировать контент.»

❌ «Представьте, что вы ведёте блог о путешествиях...»
✅ «AI-редактор автоматически добавляет благодарность автору и хештег проекта — не нужно делать это вручную.»

## Правило: не фантазировать

- Перед описанием — **найди и прочитай код**
- Если не уверен, что функция есть — **спроси разработчика**
- Описывай только то, что **реально видишь в коде**
- Если чего-то нет — так и скажи: «Этой функции в системе нет»

## Неочевидные механики (акцент в обучении)

- **Drag-and-Drop**: что можно перетащить, какой результат
- **Двойной клик**: например, создание заметки по двойному клику на пустом месте
- **Кликабельные области**: например, клик по тексту поста для сворачивания/разворачивания
