# Templates — Шаблоны записей в журнал релизов

## Технический файл (releaseX.md)

### Открытый баг

```markdown
### Краткое название проблемы

**Проблема:**  
Подробное описание бага или задачи на основе контекста разговора.

**Статус:** 🔴 В работе

---
```

### Решённый баг

```markdown
### Краткое название проблемы

**Проблема:**  
Описание проблемы.

**Причина:**  
Техническая причина возникновения (на основе анализа кода).

**Решение:**  
Подробное описание того, что было сделано для исправления.

**Файл(ы):** `путь/к/файлу.ts`

---
```

### Новая фича (технический)

```markdown
### Краткое название функции

**Описание:**  
Техническое описание нового функционала.

**Реализация:**  
Что было создано: новые файлы, эндпоинты, компоненты.

**Файл(ы):** `путь/к/файлу.ts`, `путь/к/другому.py`

---
```

## Пользовательский файл (user_rleaseX.md)

### Открытый баг

```markdown
### Краткое название проблемы

**Проблема:**  
Понятное описание для пользователя: что не работало и как это проявлялось.

**Статус:** 🔴 В работе

---
```

### Решённый баг

```markdown
### Краткое название проблемы

**Проблема:**  
Понятное описание для пользователя.

**Решение:**  
Что исправили и как это теперь работает для пользователя (без технических деталей).

---
```

### Новая фича (пользовательский)

```markdown
### Краткое название функции

**Что добавили:**  
Короткое описание новой возможности простыми словами.

**Как использовать:**  
Пошаговая инструкция, что нужно сделать в интерфейсе.

**Что теперь можно:**  
Что стало возможно сделать благодаря функции.

---
```

## Секции в файлах

Каждый файл релиза содержит секции:

```markdown
# Релиз X — Описание

## 🆕 Новые возможности
(записи фич)

## 🔴 Открытые задачи
(записи багов в работе)

## ✅ Решённые задачи
(записи решённых багов)
```

## Правила

1. **Технический файл** — подробности: причина, решение, файлы, технический язык
2. **Пользовательский файл** — простой язык, без жаргона, фокус на «что сломалось / что починили»
3. **Парность** — технический и пользовательский файлы всегда создаются и обновляются вместе
4. **При переносе из 🔴 в ✅** — удалить запись из секции «Открытые» полностью, вставить в «Решённые» с дополненным шаблоном
