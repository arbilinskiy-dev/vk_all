# Скилл: bitrix24

## Описание

Битрикс24 - полный доступ к CRM, задачам, контактам, сделкам, лидам, компаниям, сотрудникам через REST API (webhook).

## Ключевые сущности

### Чат обратной связи «Сервис автоматизаций»
- **dialog_id:** `chat34984`
- **chat_id:** `34984`
- **group_id (проект):** `500`
- **Назначение:** Обратная связь от команды (менеджеров, сотрудников) по проблемам, багам, пожеланиям в нашем сервисе VK Content Planner (он же «Сервис автоматизаций»). Сюда приходят баг-репорты, запросы фич, скриншоты проблем.
- **Задачи с group_id=500** — это задачи того же проекта, созданные из этого чата или вручную. Большинство задач на Андрея Билинского (ID=1).

### Рабочий цикл: Битрикс24 → Код → Битрикс24

Типичный флоу работы с задачами/проблемами из этого чата:

1. **Изучить задачу/проблему** — собрать контекст из Битрикс24
2. **Исправить/доработать** — написать код в этом репозитории (VK Content Planner)
3. **Отписаться в чат** — сообщить о решении/статусе в чат «Сервис автоматизаций»

## Процедура: Работа с задачами Сервиса автоматизаций

### Когда триггерится
- Пользователь просит взять задачу из Битрикс24
- Просит посмотреть что нового в чате
- Говорит «задачи», «баги», «что там в битриксе», «проблемы сервиса»
- Называет конкретный ID задачи

### Шаг 1. Получить список актуальных задач
```
b24_get_tasks(responsible_id=1, limit=50)
```
Отфильтровать задачи с `group_id=500` (проект «Сервис автоматизаций»), показать активные (status != 5).

### Шаг 2. Собрать полный контекст задачи

Для конкретной задачи — два варианта:

**Вариант А — Задача создана из чата** (в описании есть `IM_DIALOG=chat34984`):
```
b24_get_dev_task_context(task_id=XXXXX, messages_count=15)
```
Этот вызов:
- Вытаскивает задачу со всеми полями
- Парсит ссылку на чат из описания
- Загружает 15 сообщений вокруг исходного
- Фильтрует системные уведомления
- Собирает все вложения задачи с download URL
- Достаёт комментарии и чеклист

**Вариант Б — Задача без привязки к чату:**
```
b24_get_task_full_context(task_id=XXXXX)
```
Плюс вручную посмотреть последние сообщения чата:
```
b24_get_dialog_messages(dialog_id="chat34984", limit=30)
```

### Шаг 3. Проанализировать вложения
Для каждого файла из `task_attachments` и `chat_attachments`:
1. Открыть `download_url` через Playwright (`browser_navigate`)
2. Сделать скриншот (`browser_take_screenshot`)
3. Проанализировать содержимое (скриншот UI, лог ошибки, документ)

### Шаг 4. Скомпилировать контекст для разработки
Собрать итоговое описание:
- **ЧТО:** суть бага/фичи из описания задачи + сообщений чата
- **КТО СООБЩИЛ:** автор сообщения в чате (менеджер)
- **КОНТЕКСТ:** релевантные сообщения из чата (кто, что написал)
- **ВИЗУАЛ:** описание каждого скриншота/вложения
- **ЧЕК-ЛИСТ:** что нужно сделать (если есть)
- **ГДЕ В КОДЕ:** определить затронутые модули в репозитории (features/, backend_python/, services/)

### Шаг 5. Исправить / Доработать
Работаем с кодом в репозитории VK Content Planner:
- Фронтенд: `features/`, `shared/`, `services/`
- Бэкенд: `backend_python/routers/`, `backend_python/services/`, `backend_python/crud/`
- Используем скилл `bug-killer` для диагностики, `impact-analysis` для оценки влияния

### Шаг 6. Правильная коммуникация после фикса (обязательно 3 канала)
После исправления делаем отписку в три места:

1. **Комментарий в задаче (технический контекст для команды)**
```
b24_add_task_comment(task_id=XXXXX, message="<тех.причина + что изменили + что проверить>")
```

2. **Ответ пользователю в чате на его исходное сообщение (юзер-френдли)**
```
b24_send_message(
	dialog_id="chat34984",
	message="<простое объяснение без тех.жаргона>",
	reply_to_message_id=3765788
)
```

3. **Короткая фиксация в чате задачи (статус + ссылка на задачу)**
```
b24_send_message(dialog_id="chat34984", message="✅ Задача #XXXXX выполнена. Проверьте, пожалуйста, сценарий...")
```

Если нужно сделать всё за один вызов:
```
b24_report_task_completion(
	task_id=XXXXX,
	user_message="<юзер-френдли ответ>",
	technical_note="<техническая заметка>",
	user_dialog_id="chat34984",
	user_reply_message_id=3765788,
	task_chat_dialog_id="chat34984",
	task_chat_message="<краткий статус в чате задачи>"
)
```

### Шаблон user-friendly сообщения (для обычных пользователей)
- Начать с результата: «Исправили, можно проверять».
- Коротко объяснить причину простыми словами.
- Дать 2-3 шага, что именно протестировать.
- Попросить в случае повторения написать в чат задачи и дать номер/ссылку.

Пример:
```
✅ Всё исправили по задаче #214604.

Почему так происходило: после сохранения система не запоминала выбранный режим и при повторном открытии показывала значение по умолчанию.

Что проверить:
1) Выберите режим «минимум»
2) Нажмите «Сохранить»
3) Откройте настройки снова и проверьте, что режим сохранился

Если что-то снова пойдёт не так, напишите, пожалуйста, в чат задачи #214604 (лучше с коротким скриншотом).
```

### Шаблон технического комментария в задаче
- Симптом
- Root cause
- Что изменено (файл/модуль)
- Что протестировано
- Результат ретеста от пользователей (когда появится)

Пример:
```
[FIX] Исправлен сброс targetCountMode в exact после повторного открытия страницы.

Root cause:
- На загрузке настроек поле targetCountMode не маппилось в loadedSettings.

Изменения:
- features/automations/reviews-contest/ReviewsContestPage.tsx: добавлен targetCountMode в loadedSettings.
- services/api/automations.api.ts: DTO расширен полем targetCountMode.

Проверка:
- frontend build успешен
- регресс в изменённых файлах не выявлен
```

### Шаг 7. (опционально) Закрыть задачу
Если задача полностью решена:
```
b24_complete_task(task_id=XXXXX)
```

## Процедура: Мониторинг чата

Когда пользователь просит «что нового в чате», «проверь чат», «последние проблемы»:

```
b24_get_dialog_messages(dialog_id="chat34984", limit=30)
```
Показать последние сообщения, выделив:
- Баг-репорты и проблемы (красные флаги)
- Запросы на фичи
- Вопросы, ждущие ответа

## Процедура: Анализ покрытия чата задачами (чтобы не терять задачи)

Когда пользователь просит: «проверь, всё ли заведено в задачи», «что потеряли»,
«что выполнено / не выполнено по чату».

Базовый вызов:
```
b24_analyze_chat_task_coverage(
	dialog_id="chat34984",
	group_id=500,
	messages_limit=100,
	tasks_limit=200
)
```

Что возвращает анализ:
- `summary` — агрегаты (сколько обращений, сколько задач, открытые/выполненные, потери)
- `user_messages_without_task` — сообщения пользователей, по которым задача не найдена
- `open_tasks` — активные задачи
- `completed_tasks` — выполненные задачи
- `tasks_without_chat_link` — задачи без явной ссылки на сообщение чата

Правило действий по результату:
1. Если есть `user_messages_without_task` — завести задачи в первую очередь.
2. Если много `tasks_without_chat_link` — добавить в описание задачи ссылку на исходное сообщение.
3. По `open_tasks` — дать короткий статус в чат (что в работе, что на тесте).
4. По `completed_tasks` — проверить, что есть user-friendly отписка пользователю.

## Процедура: Общая CRM

1. Определить намерение: CRM (лиды/сделки/контакты/компании), задачи, активити, сотрудники
2. Вызвать соответствующий MCP-инструмент
3. Отформатировать результат в читаемую сводку
4. При создании/обновлении — подтвердить результат

## Примеры запросов

### Сервис автоматизаций (основной рабочий поток)
- «покажи задачи» → `b24_get_tasks(responsible_id=1)` → фильтр по group_id=500
- «что нового в чате» → `b24_get_dialog_messages(dialog_id="chat34984", limit=30)`
- «разбери задачу 214604» → `b24_get_dev_task_context(task_id=214604)` → анализ → правка кода → `b24_add_task_comment` + `b24_send_message(..., reply_to_message_id=...)`
- «отпишись пользователю на его сообщение» → `b24_send_message(dialog_id="chat34984", message="...", reply_to_message_id=3765788)`
- «сделай полный отчёт по задаче» → `b24_report_task_completion(...)`
- «проверь, не потеряли ли задачи из чата» → `b24_analyze_chat_task_coverage(dialog_id="chat34984", group_id=500)`
- «закрой задачу 214284» → `b24_complete_task(task_id=214284)`
- «контекст вокруг задачи 214430» → `b24_get_task_chat_context(task_id=214430)`

### CRM
- «покажи новые лиды» → `b24_get_leads(status_id="NEW")`
- «сделки в работе» → `b24_get_deals(closed=False)`
- «создай лид для ООО Ромашка» → `b24_add_lead(title="ООО Ромашка", ...)`
- «найди в CRM Иванов» → `b24_search_crm(query="Иванов")`

### Диск и файлы
- «файлы задачи 214284» → `b24_get_task_full_context(task_id=214284)` → `b24_get_attached_object(object_id)` → DOWNLOAD_URL
- «что на диске» → `b24_get_storage_list()` → `b24_get_folder_children(folder_id)`

## Особенности

- **Задачи из чата**: Если задача создана из мессенджера, в описании есть BB-код `IM_DIALOG=chatXXXX&IM_MESSAGE=YYYYYYY`. Метод `b24_get_task_chat_context` автоматически парсит его и вытягивает контекст переписки.
- **Файлы задач**: Вложения хранятся в поле `ufTaskWebdavFiles` как массив attachedId. Для получения информации о файле и ссылки на скачивание — `b24_get_attached_object(attachedId)`. Для полной инфы о файле на диске — `b24_get_file_info(OBJECT_ID из attached)`.
- **Webhook permissions**: CRM, Задачи, Пользователи, Структура компании, Чат и уведомления (im), Диск (disk).
- **ID пользователя Андрея Билинского:** `1` (основатель, основной исполнитель).
- **Проект «Сервис автоматизаций»:** group_id=`500`, чат=`chat34984` — основная точка входа для задач разработки.
