# Скилл: Мастер-оркестратор создания Автор-Скилла

## Описание

Этот скилл координирует полный цикл создания `author-writer` скилла:
1. Запускает 6 экспертных анализаторов параллельно через саб-агентов
2. Собирает результаты всех анализов
3. Синтезирует финальный `author-writer` SKILL.md

## Входные данные

- Тексты автора: `backend_python/data/author_texts_for_analysis.json`
- Скиллы-анализаторы: `.agents/skills/author-analyze-*/SKILL.md`

## Выходные данные

- Результаты анализа: `backend_python/data/author_analysis/*.json` (6 файлов)
- Финальный скилл: `.agents/skills/author-writer/SKILL.md`

---

## Процедура

### Шаг 0: Подготовка

Создать папку для результатов анализа (если не существует):
```
backend_python/data/author_analysis/
```

### Шаг 1: Запуск анализаторов (⚡ ПАРАЛЛЕЛЬНО — 3 батча по 2)

Запустить 6 саб-агентов. Каждый саб-агент:
1. Читает SKILL.md своего анализатора (инструкции)
2. Читает `author_texts_for_analysis.json` (данные)
3. Проводит анализ по инструкциям
4. Сохраняет полный результат в JSON-файл
5. Возвращает краткое резюме (`for_author_skill`)

#### Батч 1 (⚡ ПАРАЛЛЕЛЬНО):

**Саб-агент A: Психология**
```
Промпт: Прочитай файл .agents/skills/author-analyze-psychology/SKILL.md — это твои инструкции. 
Затем прочитай backend_python/data/author_texts_for_analysis.json — это тексты автора.
Выполни полный психологический анализ по инструкциям в SKILL.md.
Сохрани результат в backend_python/data/author_analysis/psychology.json.
Верни ТОЛЬКО содержимое поля for_author_skill (компактная выжимка для автор-скилла).
Язык ответа: русский.
```

**Саб-агент B: Стиль письма**
```
Промпт: Прочитай файл .agents/skills/author-analyze-writing/SKILL.md — это твои инструкции.
Затем прочитай backend_python/data/author_texts_for_analysis.json.
Выполни полный анализ стиля письма по инструкциям.
Сохрани результат в backend_python/data/author_analysis/writing_style.json.
Верни ТОЛЬКО содержимое поля for_author_skill.
Язык ответа: русский.
```

#### Батч 2 (⚡ ПАРАЛЛЕЛЬНО):

**Саб-агент C: Риторика**
```
Промпт: Прочитай .agents/skills/author-analyze-rhetoric/SKILL.md.
Изучи backend_python/data/author_texts_for_analysis.json.
Выполни анализ риторики и нарративных паттернов.
Сохрани в backend_python/data/author_analysis/rhetoric.json.
Верни ТОЛЬКО for_author_skill.
Язык: русский.
```

**Саб-агент D: Влияние и маркетинг**
```
Промпт: Прочитай .agents/skills/author-analyze-influence/SKILL.md.
Изучи backend_python/data/author_texts_for_analysis.json.
Выполни анализ методов влияния и маркетинга.
Сохрани в backend_python/data/author_analysis/influence.json.
Верни ТОЛЬКО for_author_skill.
Язык: русский.
```

#### Батч 3 (⚡ ПАРАЛЛЕЛЬНО):

**Саб-агент E: Ценности**
```
Промпт: Прочитай .agents/skills/author-analyze-values/SKILL.md.
Изучи backend_python/data/author_texts_for_analysis.json.
Выполни анализ ценностей и мировоззрения.
Сохрани в backend_python/data/author_analysis/values.json.
Верни ТОЛЬКО for_author_skill.
Язык: русский.
```

**Саб-агент F: Структура**
```
Промпт: Прочитай .agents/skills/author-analyze-structure/SKILL.md.
Изучи backend_python/data/author_texts_for_analysis.json.
Выполни структурный анализ постов.
Сохрани в backend_python/data/author_analysis/structure.json.
Верни ТОЛЬКО for_author_skill.
Язык: русский.
```

### Шаг 2: Сбор результатов

После завершения всех 6 саб-агентов, собрать:
- 6 значений `for_author_skill` (краткие выжимки от каждого эксперта)
- Проверить, что все 6 файлов JSON созданы в `backend_python/data/author_analysis/`

### Шаг 3: Генерация author-writer SKILL.md

Прочитать шаблон: `.agents/skills/author-writer/SKILL_TEMPLATE.md`

Заполнить шаблон данными из 6 анализов:
- Секция `## Психологический портрет` ← psychology.for_author_skill
- Секция `## Стиль письма` ← writing_style.for_author_skill
- Секция `## Риторические приёмы` ← rhetoric.for_author_skill
- Секция `## Методы влияния` ← influence.for_author_skill
- Секция `## Ценности и мировоззрение` ← values.for_author_skill
- Секция `## Структура постов` ← structure.for_author_skill

Дополнительно сгенерировать:
- Секция `## Быстрый старт` — 10 главных правил
- Секция `## Чеклист перед публикацией` — проверочные вопросы
- Секция `## Anti-паттерны` — чего автор НИКОГДА не делает

Сохранить финальный скилл в `.agents/skills/author-writer/SKILL.md`.

### Шаг 4: Валидация

- Прочитать готовый SKILL.md
- Убедиться, что все 6 секций заполнены
- Убедиться, что скилл actionable (содержит конкретные инструкции, а не абстракции)
- При необходимости: дочитать полные JSON-файлы анализов для дополнительных деталей

### Шаг 5: Отчёт пользователю

Вывести:
- Статус каждого анализатора (✅/❌)
- Ключевые инсайты от каждого (1-2 предложения)
- Путь к готовому скиллу
- Размер каждого файла анализа
- Инструкция по использованию author-writer

---

## Важные правила

1. **Максимум 3 саб-агента одновременно** (ограничение системы)
2. **Каждый саб-агент получает ПОЛНЫЙ промпт** — контекст предыдущего диалога недоступен
3. **Файл author_texts_for_analysis.json большой (~2.7MB)** — каждый саб-агент читает его сам
4. **Не останавливаться если один анализатор упал** — запустить остальные, потом повторить упавший
5. **Результат должен быть КОНКРЕТНЫМ** — не «автор пишет хорошо», а «автор начинает 70% постов с нумерации дней, использует парцелляцию, средняя длина предложения 12 слов»
