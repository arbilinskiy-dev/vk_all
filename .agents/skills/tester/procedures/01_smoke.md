# Процедура: Smoke-тест

**Триггер:** «проверь что не сломалось», «smoke-тест», «тест после рефакторинга»
**Время:** ~10 секунд
**Приоритет:** 🔴 P0

## TODO-шаблон

При запуске этой процедуры — создать TODO-лист:
1. [ ] Определить область (папка / модуль / весь проект)
2. [ ] Проверка Python-импортов целевой области
3. [ ] Проверка эндпоинтов (убедиться что uvicorn запущен)
4. [ ] Проверка TypeScript (если затронут фронтенд)
5. [ ] Сводка результатов

## Что проверяет

1. **Импорты** — все .py файлы в целевой папке импортируются без ошибок
2. **Эндпоинты** — все роуты отвечают (не 500)
3. **TypeScript** — `tsc --noEmit` для фронтенда (если затронут)

## Алгоритм

1. Определить область (папка / модуль / весь проект)
2. Запустить проверку импортов → `scripts/check_imports.py <путь>`
3. Запустить smoke-тест эндпоинтов → `scripts/smoke_test.py` (убедиться что uvicorn запущен на localhost:8000)
4. Если затронут фронтенд → запустить `tsc --noEmit`
5. Собрать результаты → вывести вердикт (кратко: ✅/❌ по каждому пункту)

## Делегирование саб-агентам (⚡ параллельно)

| Саб-агент | Задача |
|---|---|
| A: Python-импорты | Запустить `scripts/check_imports.py` или проверить import-ы вручную |
| B: TypeScript | Запустить `tsc --noEmit` или `get_errors` |

Эндпоинты проверяются в основном потоке (нужен запущенный uvicorn).

## Связанные ресурсы

- `scripts/smoke_test.py` — автономный smoke-тест эндпоинтов
- `scripts/check_imports.py` — проверка импортов Python-модулей
