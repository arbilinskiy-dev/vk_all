## Что проверяет

1. **Auth boundary** — каждый эндпоинт с `Depends(get_current_user)` возвращает 401 без токена
2. **Публичные эндпоинты** — только `/version`, `/auth/login`, `/auth/callback` доступны без токена
3. **IDOR** — пользователь A не может получить данные пользователя B
4. **Privilege escalation** — обычный пользователь не может выполнить admin-действие
5. **Истёкший токен** — возвращает 401

## Классификация

| Категория | Проверка |
|---|---|
| Защищённые (auth) | GET без токена → 401 |
| Публичные | GET без токена → НЕ 401 |
| Admin-only | GET с user-токеном → 403 |
| Owner-only | GET с чужим user_id → 403/404 |
