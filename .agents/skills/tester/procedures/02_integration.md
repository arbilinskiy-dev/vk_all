**Триггер:** «покрой тестами», «тест бизнес-логики», «интеграционные тесты»
**Время:** 1-5 минут
**Приоритет:** 🟡 P1
**⚡ Параллельность:** БЕЗОПАСНО с frontend-тестами. Пишет в `tests/integration/`. В СЛОЕ 3 — Саб-агент A параллельно с contract (Саб-агент B).

## Алгоритм

1. Определить область тестирования
2. Прочитать код целевого модуля (роутер + сервис + crud + схемы)
3. Составить список тест-кейсов:
   - CRUD-операции (создать → прочитать → обновить → удалить)
   - Валидация (невалидные данные → 422)
   - Граничные случаи (пустые данные, несуществующие ID)
   - Авторизация (без токена → 401, чужие данные → 403)
4. Написать тесты
5. Запустить и прочитать вывод

## Делегирование

Прочитать стек модуля через саб-агента:
> «Прочитай бэкенд-стек для {модуль}: routers/{модуль}.py, services/{модуль}.py, crud/{модуль}.py, schemas/{модуль}.py. Верни: эндпоинты, типы данных, SQL-запросы, потенциальные N+1»
