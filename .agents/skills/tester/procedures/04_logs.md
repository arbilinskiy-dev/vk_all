# Процедура: Чтение и анализ логов

**Триггер:** «посмотри логи», «что в логах», «прочитай логи»
**Время:** 1-3 минуты

## TODO-шаблон

1. [ ] Определить источник логов (локальный бэкенд / предпрод / фронт)
2. [ ] Прочитать логи
3. [ ] Найти ERROR, WARNING, Traceback
4. [ ] Проанализировать → вердикт

## Локальные логи (бэкенд)

1. Прочитать вывод терминала uvicorn (`get_terminal_output`)
2. Найти ERROR, WARNING, Traceback
3. Если есть файл логов → прочитать последние 100 строк

## Логи предпрода

1. Прочитать `shared/config.ts` → взять URL предпрода
2. Запросить эндпоинт здоровья: `GET /api/health`
3. Запросить проблемный эндпоинт через curl/httpx
4. Проанализировать ответ (статус, тело, заголовки)

## Фронтенд-логи

1. `tsc --noEmit` — ошибки типов
2. `npm run build` — ошибки сборки
3. Если описана ошибка в консоли браузера → проанализировать
