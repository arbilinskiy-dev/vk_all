# Процедура: Тест конкретной области

**Триггер:** «протестируй папку X», «проверь модуль Y», «протестируй функционал Z»
**Время:** зависит от выбранного типа

## TODO-шаблон

1. [ ] Определить область: папка / модуль / функционал
2. [ ] Найти связанные файлы (роутер, сервис, crud, компоненты)
3. [ ] Выбрать тип проверки (smoke / integration / целевой / полный)
4. [ ] Выполнить выбранный тип → запустить соответствующую процедуру
5. [ ] Собрать результаты → вердикт

## Маршрутизация

| Ситуация | Какую процедуру запустить |
|---|---|
| После рефакторинга | `01_smoke.md` |
| Новый функционал | `02_integration.md` |
| Подозрительный баг | Целевой тест + `04_logs.md` |
| «Покрой тестами» | Полный набор: smoke → integration → api-lint |

## Как найти файлы

- Папка → `Get-ChildItem -Recurse *.py / *.tsx`
- Модуль → найти роутер, сервис, crud, схемы по имени
- Функционал (описание) → `grep_search` по ключевым словам
