# Процедура: API-lint (проверка auth headers)

**Триггер:** «проверь fetch вызовы», «lint API», «проверь auth headers» + автоматически при каждом smoke-тесте
**Время:** ~5 секунд
**Приоритет:** 🔴 P0
**⚡ Параллельность:** БЕЗОПАСНО — read-only, не пишет файлы. В СЛОЕ 1 запускается как Саб-агент C параллельно с imports и tsc.

**Предыстория:** В марте 2026 обнаружен системный баг — 11 файлов (55 вызовов `fetch()`) в `services/api/` не отправляли `X-Session-Token`. Этот тест предотвращает повторение.

## TODO-шаблон

1. [ ] Получить список файлов `services/api/*.ts`
2. [ ] Для каждого файла: найти все `fetch()` вызовы
3. [ ] Проверить наличие `getAuthHeaders` или `callApi` в блоке ±4 строки
4. [ ] Исключить `management.api.ts` (публичный `/version`)
5. [ ] Вывести отчёт: файл, строка, есть_ли_auth

## Что проверяет

1. Все `fetch()` в `services/api/*.ts` используют `getAuthHeaders()` или `callApi()`
2. Ни один `fetch()` не содержит хардкоженные заголовки без токена
3. FormData-загрузки используют `getAuthHeaders(false)` (без Content-Type)
4. Исключения: `management.api.ts` → `/version` (публичный эндпоинт)

## Алгоритм

1. Получить список всех файлов: `services/api/*.ts`
2. Для каждого файла:
   - Найти все строки с `fetch(`
   - Проверить блок ±4 строки на наличие `getAuthHeaders` или `callApi`
   - Исключить `management.api.ts`
   - Если найден fetch БЕЗ auth → FAIL
3. Вывести отчёт
