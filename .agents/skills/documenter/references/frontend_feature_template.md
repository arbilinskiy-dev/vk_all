# Шаблон документации фронтенд-фичи

## Формат файла: `.md`
## Размещение: `docs/frontend/features/<название>/`

---

```markdown
# <Название фичи>

## Обзор

Краткое описание: что видит пользователь, для чего нужна.

## Расположение в интерфейсе

Где найти: путь в навигации, какая страница, какой раздел.

## Структура файлов

```
features/<name>/
  components/
    <MainComponent>.tsx        ← Основной компонент
    <SubComponent>.tsx         ← Вспомогательные
    modals/
      <Modal>.tsx              ← Модальные окна
  hooks/
    use<Feature>.ts            ← Основной хук логики
    use<SubFeature>.ts         ← Дополнительные хуки
  types/
    index.ts                   ← Типы
  utils/
    <helpers>.ts               ← Утилиты
```

## Компоненты

### `<MainComponent>`

**Назначение:** Что отображает и делает.

**Props:**
| Prop | Тип | Описание |
|------|-----|----------|
| `projectId` | number | ID проекта |
| `onClose` | () => void | Закрытие |

**Состояния:**
- Загрузка (skeleton)
- Пустой список
- С данными
- Ошибка

### `<Modal>`

**Открывается:** При каком действии.
**Закрывается:** Крестик / Escape / клик вне.

## Хуки

### `use<Feature>`

**Назначение:** Основная логика фичи.

**Возвращает:**
| Поле | Тип | Описание |
|------|-----|----------|
| `items` | Item[] | Список элементов |
| `isLoading` | boolean | Загрузка |
| `create` | (data) => void | Создание |
| `update` | (id, data) => void | Обновление |
| `delete` | (id) => void | Удаление |

**API-вызовы:**
- `GET /api/<path>` — загрузка
- `POST /api/<path>` — создание
- ...

## Взаимодействие с пользователем

### Основные сценарии

1. **Просмотр** — пользователь заходит → видит список → ...
2. **Создание** — нажимает кнопку → заполняет форму → сохраняет
3. **Редактирование** — кликает на элемент → изменяет → сохраняет
4. **Удаление** — нажимает удалить → подтверждает → удалено

### Неочевидные механики

- Drag-and-Drop (если есть)
- Двойной клик
- Горячие клавиши
- Автосохранение

## Зависимости

- Внешние библиотеки
```
