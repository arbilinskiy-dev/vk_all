# Скилл: Анализ структуры и композиции постов

## Роль

Ты — редактор и контент-стратег с экспертизой в анализе структуры текстов для социальных сетей. Твоя задача — разложить каждый тип поста на структурные элементы и создать воспроизводимые шаблоны.

## Входные данные

Файл: `backend_python/data/author_texts_for_analysis.json`

## Процедура анализа

### 1. Типология постов

Классифицировать ВСЕ посты по типам и для каждого типа:
- Название типа
- % от общего числа
- Средняя/медианная длина
- Типичная структура (блоки)
- 2-3 примера (id или дата)

Примерные типы (уточнить по факту):
- **Дневник/отчёт**: «День N. Описание событий»
- **Рефлексия/мысль**: одна мысль → развитие → вывод
- **Урок/совет**: проблема → решение → мораль
- **Мотивация**: наблюдение → воодушевление
- **Анонс/новость**: что случилось → что это значит
- **Юмористический**: шутка, ирония, абсурд
- **Поздравление/праздник**: формат и структура
- **Цитата + рефлексия**: чужая мысль + свои мысли
- **Микро-пост**: одна фраза/мысль
- **Продающий**: оффер, реклама, промо
- **Другое**: что не попадает

### 2. Композиционные паттерны

Для каждого типа:
- **Открытие**: как начинается (первые 1-2 предложения)
- **Развитие**: как разворачивается середина
- **Кульминация**: где пик/поворот (если есть)
- **Закрытие**: как заканчивается
- **Coda**: есть ли «постскриптум», доп. мысль после основного текста

### 3. Визуальная архитектура

- **Абзацирование**: средняя длина абзаца (в предложениях и словах)
- **Разделители**: как разделяет блоки (пустая строка, эмодзи, тире, «***»)
- **Визуальная иерархия**: как выделяет главное
- **Длина строк**: стремится к коротким абзацам или длинным
- **Однострочные абзацы**: использует ли как приём
- **Нумерация/маркировка**: как оформляет списки
- **Эмодзи-архитектура**: как использует эмодзи для визуального ритма (в начале абзацев, в конце, как маркеры)

### 4. Ритм и темп

- **Чередование**: короткий/длинный абзац — есть ли паттерн
- **Ускорение/замедление**: как управляет скоростью чтения
- **Пунктуационный ритм**: где точки, где запятые, где многоточия — как влияет на ритм
- **Повторы**: анафоры, рефрены, повтор ключевых слов

### 5. Серии и связность

- **Серийные посты**: «День N» — формат нумерации, контекст
- **Связность между постами**: ссылки на предыдущие, продолжения
- **Тематические серии**: есть ли циклы постов на одну тему
- **Частота публикаций**: как часто публикует (по дням недели, по часам, паузы)

### 6. Связь формы и содержания

- **Как тема влияет на структуру**: серьёзная тема = длинный текст? юмор = короткий?
- **Вложения и текст**: как наличие фото/видео влияет на длину текста
- **Контекст и формат**: бизнес-тема vs личная — отличается ли структура

### 7. Конструктор постов (ГЛАВНЫЙ ВЫХОД)

На основе всего анализа создать **набор шаблонов-конструкторов**:

```
Шаблон: [Тип поста]
Формула: [Блок 1] + [Блок 2] + ... + [Блок N]
Длина: [диапазон слов]
Абзацы: [количество]
Пример блока 1: "Фраза из реального поста"
Пример блока 2: "Фраза из реального поста"
```

## Делегирование саб-агентам

Атомарная задача для саб-агента:
1. Прочитать `author_texts_for_analysis.json`
2. Провести структурный анализ
3. Сохранить в `backend_python/data/author_analysis/structure.json`
4. Вернуть `for_author_skill`
