# UI/UX Рефакторинг

Этот скилл выполняет полный аудит указанной области интерфейса по дизайн-системе проекта и автоматически исправляет нарушения.

## Процедура аудита

При активации скилла выполнить строго по порядку:

### Шаг 1: Определить область проверки

Спросить пользователя (если не указано явно): какой файл, компонент или страницу проверить.

### Шаг 2: Прочитать исходный код

Прочитать все файлы указанной области: `.tsx` компоненты, связанные хуки, стили.

### Шаг 3: Загрузить эталоны

Прочитать файл `references/checklist.md` — полный чеклист проверки из 12 пунктов.  
При необходимости деталей по конкретному пункту — читать соответствующий файл из `references/`:

| Пункт чеклиста | Файл с эталоном |
|---|---|
| Скроллбары | `references/scrollbars.md` |
| Кнопки | `references/buttons.md` |
| Тумблеры | `references/toggles.md` |
| Табы / Навигация | `references/tabs.md` |
| Счётчики | `references/counters.md` |
| Уведомления / Модалки | `references/notifications.md` |
| Кнопки-иконки (SVG) | `references/icon_buttons.md` |
| Изображения / Фото | `references/images.md` |
| Цвета и захардкоженные значения | `references/colors.md` |
| Приоритет компонентов | `references/component_priority.md` |
| Общие запреты | `references/prohibitions.md` |
| Текстовый редактор | `references/text_editor.md` |
| Частные переменные (Specific Variables) | `references/specific_variables.md` |
| Анимации и плавные переходы | `references/animations.md` |
| Поле поиска (Search Input) | `references/search_input.md` |
| Выпадающие списки (Dropdown / Select) | `references/dropdowns.md` |
| Склонение числительных (Pluralization) | `references/pluralization.md` |
| Скелетоны загрузки (Cascade Reveal) | `references/animations.md` (раздел «Скелетоны — Каскадное превращение») |
| Дашборд статистики (AnimatedCounter, Sparkline, живое обновление) | `references/animations.md` (раздел 8 «Дашборд статистики») |
| Информационная панель (Info Panel, stale data, table-fixed, minHeight, donut) | `references/info_panel.md` |
| Сайдбар — Изоляция состояний вкладок (модули km/lists/am, утечки стейтов) | `references/sidebar_state_isolation.md` |

### Шаг 4: Провести аудит

Пройтись по каждому пункту чеклиста и проверить код на соответствие.  
Для каждого нарушения зафиксировать:
- Какой пункт нарушен
- Где в файле (строка/блок кода)
- Что должно быть по эталону

### Шаг 5: Составить отчёт

Вывести пользователю таблицу:

```
| # | Правило | Статус | Что не так | Файл:строка |
|---|---------|--------|-----------|-------------|
| 1 | Скроллбары | ✅ OK | — | — |
| 2 | Кнопки | ❌ | bg-blue-500 вместо bg-green-600 | Component.tsx:45 |
| ... |
```

### Шаг 6: Исправить нарушения

Автоматически исправить все найденные нарушения в коде. Для каждого исправления кратко указать, какое правило дизайн-системы было применено.

### Шаг 7: Финальная проверка

После всех правок — пройти чеклист ещё раз и убедиться, что нарушений не осталось.

## Важно

- НЕ трогать бизнес-логику — только визуальное соответствие дизайн-системе
- НЕ менять структуру компонента, если это не требуется правилами
- Если в `ui-kit/` или `shared/components/` есть готовый компонент — предложить замену
- Комментарии в коде — на русском языке

## Делегирование саб-агентам

Для экономии контекстного окна следующие шаги рекомендуется выполнять через `runSubagent`:

| Шаг | Что делегировать | Промпт для саб-агента |
|---|---|---|
| ⚡ Шаг 2 — Инвентаризация кода | Чтение всех файлов области (5–20 файлов) | «Прочитай все файлы в {область}. Для каждого выпиши: CSS-классы Tailwind, inline-стили, захардкоженные цвета, импорты из shared/components/, использование window.confirm/alert. Верни структурированный отчёт» |
| ⚡ Шаг 3 — Загрузка эталонов (⚡ ГЛАВНЫЙ) | Чтение 20 reference-файлов (~2000–4000 строк) | «Прочитай references/checklist.md и ВСЕ файлы из references/: scrollbars, buttons, toggles, tabs, counters, notifications, icon_buttons, images, colors, component_priority, prohibitions, text_editor, specific_variables, animations, search_input, dropdowns, pluralization, info_panel, sidebar_state_isolation. Для каждого верни: правило, эталонный класс/компонент, запрещённые паттерны. Формат — компактная таблица» |
| Шаг 4 — Аудит | Перекрёстное сопоставление кода и эталонов | «Вот отчёт по коду: {отчёт}. Вот эталоны: {конспект}. Сопоставь каждый пункт чеклиста с кодом. Для нарушений укажи: пункт, файл:строка, текущий код, ожидаемый. Формат — таблица» |
| Шаг 7 — Финальная проверка | Повторный проход по исправленным файлам | «Прочитай файлы {список}. Проверь по чеклисту: {конспект эталонов}. Если нарушений нет — верни '✅ Все пункты пройдены'. Иначе — файл:строка и проблему» |

**⚡ ПАРАЛЛЕЛЬНО:** «Инвентаризация кода» (Шаг 2) + «Загрузка эталонов» (Шаг 3) — запускать **одновременно** (один читает код области, другой — reference-файлы). Это самая ценная параллелизация в этом скилле — экономит ~50% времени на подготовку. Шаг 4 (Аудит) запускать только после завершения обоих.
