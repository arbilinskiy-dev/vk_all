# Приоритет компонентов

**Правило:** При выборе реализации элемента интерфейса следовать строгой иерархии приоритетов.

## Иерархия приоритетов

1. **Компоненты из `ui-kit/` или `shared/components/`** — если существует готовый компонент, использовать его
2. **Глобальные стили из `index.css`** — если определён глобальный класс (например, `.custom-scrollbar`)
3. **Локальные стили** — только если нет глобального решения

## Процедура проверки

При аудите для каждого UI-элемента:

1. **Проверить `shared/components/`** — есть ли готовый компонент?
   - `shared/components/modals/ConfirmationModal.tsx` — для подтверждений
   - Другие переиспользуемые компоненты
   
2. **Проверить `ui-kit/`** — есть ли компонент в UI-ките?

3. **Проверить `index.css`** — есть ли глобальный класс?
   - `.custom-scrollbar` — для прокручиваемых контейнеров

4. **Только если ничего не найдено** — локальная реализация допустима

## Типичные нарушения

| Нарушение | Должно быть |
|---|---|
| Свой `confirm()` диалог | `ConfirmationModal` из `shared/components/modals/` |
| Кастомный скроллбар через CSS | `.custom-scrollbar` из `index.css` |
| Дублирование логики toast | `showAppToast()` или `toastBridge` |
| Новый компонент кнопки | Стандартные классы из дизайн-системы |
