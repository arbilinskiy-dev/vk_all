# Частные переменные (Specific Variables) в текстовом редакторе

**Правило:** Некоторые текстовые редакторы (шаблоны автоматизаций, конкурсов, рассылок) поддерживают **частные переменные** — плейсхолдеры, которые работают **исключительно** в конкретном поле и заменяются на реальные значения при выполнении сценария.

## Примеры частных переменных

| Контекст | Переменная | Описание |
|---|---|---|
| Конкурс отзывов — Комментарий регистрации | `{number}` | Порядковый номер участника |
| Конкурс отзывов — ЛС победителю | `{promo_code}` | Выигрышный промокод |
| Конкурс отзывов — ЛС победителю | `{description}` | Описание приза из базы промокодов |
| Конкурс отзывов — ЛС победителю | `{user_name}` | Имя победителя |
| Конкурс отзывов — Ошибка отправки | `{user_name}` | Имя победителя для упоминания |
| Конкурс отзывов — Пост с итогами | `{winners_list}` | Список победителей (Имя + Номер) |

## Расположение в редакторе

Частные переменные отображаются **внутри панели переменных** (раскрываемой по кнопке ⚙️), в виде **отдельной секции перед** стандартными секциями `VariablesSelector` (глобальные, базовые, конструкции, переменные проекта).

### Визуальная структура

```
┌──────────────────────────────────────────────────────────────────────┐
│  LABEL                               [⚙️ Переменные] | [😊] ↩ ↪    │  ← Тулбар
├──────────────────────────────────────────────────────────────────────┤
│  ▼ Панель переменных (плавная анимация keep-in-DOM)                 │
│                                                                      │
│  ЧАСТНЫЕ ПЕРЕМЕННЫЕ                                                  │
│  Работают исключительно в этом поле. При отправке заменяются...     │
│  (Промокод) (Описание приза) (Имя пользователя)                     │  ← rounded-full индиго
│                                                                      │
│  ГЛОБАЛЬНЫЕ ПЕРЕМЕННЫЕ                                               │
│  ...                                                                 │  ← VariablesSelector
│  БАЗОВЫЕ ПЕРЕМЕННЫЕ                                                  │
│  ...                                                                 │
├──────────────────────────────────────────────────────────────────────┤
│  textarea                                                            │
└──────────────────────────────────────────────────────────────────────┘
```

### Анимация панели переменных

При нажатии на кнопку ⚙️ панель со всеми переменными (частные + проектные) **плавно раскрывается и сворачивается** через CSS-transition по паттерну `keep-in-DOM`:

```tsx
const [variablesEverShown, setVariablesEverShown] = useState(false);

<div className={`overflow-hidden transition-all duration-300 ease-out ${
    showVariables ? 'max-h-[400px] opacity-100' : 'max-h-0 opacity-0'
}`}>
    {variablesEverShown && (
        <div className="p-3 bg-gray-50 border-b border-gray-200 space-y-4">
            {/* Секция 1: Частные переменные */}
            {specificVariables?.length > 0 && (
                <div>
                    <h4 className="text-xs font-semibold text-gray-500 uppercase tracking-wider mb-2">
                        Частные переменные
                    </h4>
                    <p className="text-xs text-gray-400 mb-2">
                        Работают исключительно в этом поле. При отправке заменяются на реальные значения.
                    </p>
                    <div className="flex flex-wrap gap-2">
                        {specificVariables.map(v => (
                            <button ...>{v.name}</button>
                        ))}
                    </div>
                </div>
            )}

            {/* Секция 2+: VariablesSelector (глобальные, базовые, конструкции, проекта) */}
            <VariablesSelector ... />
        </div>
    )}
</div>
```

**Принцип работы:**
1. **Первое открытие:** `variablesEverShown = true` → контент рендерится, transition анимирует открытие
2. **Закрытие:** контент остаётся в DOM, transition плавно схлопывает высоту и прозрачность
3. **Повторное открытие:** мгновенно, без перезагрузки данных
4. **`space-y-4`** на родительском `div` обеспечивает единообразные отступы между секциями

**Порядок элементов тулбара (слева направо):**

1. **Лейбл** (название поля) — `text-sm font-medium text-gray-700 shrink-0`
2. **Пружина** — `flex-1` (раздвигает лейбл и кнопки вправо)
3. **Кнопка переменных** ⚙️ — раскрывает панель-аккордеон
4. **Разделитель** — `w-px h-4 bg-gray-300 mx-1`
5. **Кнопка эмодзи** 😊
6. **Разделитель** — `w-px h-4 bg-gray-300 mx-1`
7. **Кнопка Undo** ↩ — `Ctrl+Z`
8. **Кнопка Redo** ↪ — `Ctrl+Shift+Z`

> **Примечание:** Второй `flex-1` между эмодзи и undo **убран** — все кнопки сгруппированы справа.

## Стиль кнопок частных переменных

```tsx
<button
    key={v.value}
    type="button"
    onClick={() => handleInsert(v.value)}
    title={`${v.description}`}
    className="px-3 py-1.5 text-xs font-medium border rounded-full transition-colors bg-indigo-50 border-indigo-200 text-indigo-700 hover:bg-indigo-100"
>
    {v.name}
</button>
```

**Ключевые характеристики:**
- **Форма:** `rounded-full` — pill-кнопки, единообразно с секциями `VariablesSelector`
- **Размер:** `px-3 py-1.5` — чуть просторнее, чем старый `px-2 py-1`
- **Фон:** `bg-indigo-50` — мягкий индиго, визуально отделён от стандартных переменных
- **Текст:** `text-indigo-700 text-xs font-medium` — читаемый
- **Бордер:** `border border-indigo-200` — подчёркивает интерактивность
- **Hover:** `hover:bg-indigo-100` — более яркий индиго при наведении
- **Контент:** Отображается **название** переменной (`v.name`): «Промокод», «Описание приза» и т.д.
- **Title:** Tooltip с описанием: `"Выигрышный промокод"`

> **Изменение:** Раньше отображался `v.value` (`{promo_code}`), теперь `v.name` (`Промокод`) — понятнее для пользователя.

## Секция частных переменных внутри аккордеона

Частные переменные оформлены как первая секция аккордеона с заголовком, шевроном, пояснением и pill-кнопками:

```tsx
{specificVariables && specificVariables.length > 0 && renderSection('specific', 'Частные переменные', (
    <>
        <p className="text-xs text-gray-400 mb-2">
            Работают исключительно в этом поле. При отправке заменяются на реальные значения.
        </p>
        <div className="flex flex-wrap gap-2">
            {specificVariables.map(v => (
                <button
                    key={v.value}
                    type="button"
                    onClick={() => handleInsert(v.value)}
                    title={v.description}
                    className="px-3 py-1.5 text-xs font-medium border rounded-full transition-colors bg-indigo-50 border-indigo-200 text-indigo-700 hover:bg-indigo-100"
                >
                    {v.name}
                </button>
            ))}
        </div>
    </>
))}
```

Заголовок и шеврон рендерятся хелпером `renderSection`. Content появляется с анимацией `max-h-60 / max-h-0`.

## Интерфейс данных частных переменных

```typescript
interface SpecificVariable {
    /** Название переменной для UI (напр. "Промокод") */
    name: string;
    /** Значение для вставки (напр. "{promo_code}") */
    value: string;
    /** Описание для tooltip (напр. "Выигрышный промокод") */
    description: string;
}
```

## Помощь/подсказка под редактором

У шаблонных полей часто есть пояснительный текст (`helpText`), объясняющий пользователю назначение поля:

```tsx
{helpText && (
    <p className="text-xs text-gray-500 mt-1.5">{helpText}</p>
)}
```

Стиль: `text-xs text-gray-500 mt-1.5` — мелкий серый текст под редактором.

## Интеграция со стандартным текстовым редактором

Частные переменные **не заменяют** стандартный функционал редактора. Они **дополняют** его:

- ✅ Undo/Redo работает для вставки частных переменных
- ✅ Автозакрытие скобок работает вокруг частных переменных
- ✅ Вставка в позицию курсора с сохранением фокуса
- ✅ Счётчик символов учитывает плейсхолдеры как обычный текст
- ✅ Emoji, переменные проекта и частные переменные работают параллельно

## Типичные ошибки

| Ошибка | Должно быть |
|---|---|
| Все секции открыты одновременно (панель огромная) | **Аккордеон**: только один раздел открыт |
| Используется `VariablesSelector` как есть | Секции рендерятся вручную через `renderSection` + `renderVarBtn` |
| Кнопки ⚙️ и 😊 в центре тулбара | Все кнопки сгруппированы **справа** (один `flex-1` перед ними) |
| Кнопки показывают `v.value` (`{promo_code}`) | Показывать `v.name` (`Промокод`) |
| Скругление `rounded` (прямоугольные) | `rounded-full` (pill-кнопки) |
| Потеря фокуса при вставке переменной | `setTimeout(() => textarea.focus(), 0)` |
| Нет tooltip с описанием | Обязательный `title` с описанием переменной |
