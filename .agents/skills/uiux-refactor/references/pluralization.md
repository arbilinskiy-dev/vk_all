# Склонение числительных (Pluralization)

## Проблема

В русском языке числительные требуют разных форм существительного:
- **1, 21, 31, 41, 51…** → `год`, `день`, `сообщение` (именительный, единственное)
- **2–4, 22–24, 32–34, 42–44…** → `года`, `дня`, `сообщения` (родительный, единственное)
- **5–20, 25–30, 35–40, 45–50…** → `лет`, `дней`, `сообщений` (родительный, множественное)

Исключение: числа 11–14 → всегда третья форма (`11 лет`, `12 дней`, `13 сообщений`).

## Примеры ошибок

```tsx
// ❌ НЕПРАВИЛЬНО — захардкожен один суффикс
<span>{age} лет</span>           // 34 лет (должно быть 34 года)
<span>{count} дней назад</span>  // 1 дней назад (должно быть 1 день)
<span>{n} сообщений</span>       // 21 сообщений (должно быть 21 сообщение)
```

## Эталонная функция

```tsx
/**
 * Универсальная плюрализация для русского языка.
 * @param n — число
 * @param forms — три формы: [именительный ед., родительный ед., родительный мн.]
 * @example plural(1,  ['год', 'года', 'лет']) → '1 год'
 * @example plural(23, ['год', 'года', 'лет']) → '23 года'
 * @example plural(35, ['год', 'года', 'лет']) → '35 лет'
 * @example plural(11, ['год', 'года', 'лет']) → '11 лет'
 */
function plural(n: number, forms: [string, string, string]): string {
    const mod10 = n % 10;
    const mod100 = n % 100;
    if (mod10 === 1 && mod100 !== 11) return `${n} ${forms[0]}`;
    if (mod10 >= 2 && mod10 <= 4 && (mod100 < 10 || mod100 >= 20)) return `${n} ${forms[1]}`;
    return `${n} ${forms[2]}`;
}
```

## Реализация в проекте

### Возраст (уже реализовано)

Файл: `features/messages/components/userInfoPanelUtils.tsx`

```tsx
/** Склонение слова «год» для возраста: 1 год, 2 года, 5 лет */
export function pluralAge(n: number): string {
    const mod10 = n % 10;
    const mod100 = n % 100;
    if (mod10 === 1 && mod100 !== 11) return `${n} год`;
    if (mod10 >= 2 && mod10 <= 4 && (mod100 < 10 || mod100 >= 20)) return `${n} года`;
    return `${n} лет`;
}
```

Использование в `UserInfoProfileTab.tsx`:
```tsx
<span>{age !== null ? pluralAge(age) : 'ДР'}</span>
```

## Частые единицы для проверки

| Единица | Формы `[ед., род.ед., род.мн.]` |
|---------|-------------------------------|
| Возраст | `['год', 'года', 'лет']` |
| Дни | `['день', 'дня', 'дней']` |
| Месяцы | `['месяц', 'месяца', 'месяцев']` |
| Сообщения | `['сообщение', 'сообщения', 'сообщений']` |
| Проекты | `['проект', 'проекта', 'проектов']` |
| Посты | `['пост', 'поста', 'постов']` |
| Товары | `['товар', 'товара', 'товаров']` |
| Фото | `['фото', 'фото', 'фото']` (не изменяется) |
| Подписчики | `['подписчик', 'подписчика', 'подписчиков']` |

## Что проверять при аудите

1. **Искать** все шаблонные строки вида `` `${count} ...` `` и `` `${n} ...` `` — проверять, не захардкожена ли одна форма
2. **Искать** конкатенации `+ ' лет'`, `+ ' дней'`, `+ ' сообщений'` и аналогичные
3. **Проверять** компоненты, которые отображают: возраст, дата подписки, давность событий, счётчики элементов
4. **Исправлять** заменой на универсальную функцию `plural(n, forms)` или специализированную (как `pluralAge`)
