# Review Contests (Конкурсы отзывов)

Автоматизация проведения конкурсов отзывов с нумерацией участников и розыгрышем призов.

## 📋 Описание

Функционал позволяет:
1. Автоматически находить посты с отзывами по ключевому слову (хештегу)
2. Нумеровать участников комментарием от имени сообщества
3. Проводить розыгрыш в назначенное время (автоматически через Post Tracker)
4. Генерировать изображение-доказательство розыгрыша
5. Выдавать призы (промокоды) победителям через ЛС
6. Публиковать пост с итогами

```
└── crud.py              # CRUD операции с БД
```

### Условия завершения (finish_condition)

| Значение | Описание |
|----------|----------|
| `count` | По достижении целевого количества участников |

| `mixed` | День настал + достигнуто целевое количество участников |

## 🔄 Полный автоматический цикл

### Триггер: Post Tracker

При наступлении времени розыгрыша, Post Tracker находит системный пост с `post_type='contest_winner'` и вызывает:

```python
contest_service.execute_scheduled_contest(db, project_id)
```

### Этап 1: Сбор участников (`collector.py`)

```
1. Проверяем свежесть кеша постов (SystemListPost)
   │
   ├─ Если данные старше 12 часов — обновляем последние 100 постов
   │
2. Ищем посты по ключевому слову и дате начала
   │
3. Фильтруем уже добавленных участников
   │
4. Находим токен администратора группы (get_admin_token_for_group)
   │
5. Запрашиваем wall.get с extended=1 и АДМИНСКИМ токеном
   │
   └─ ВАЖНО: Только админский токен возвращает post_author_data!
   │
6. Извлекаем ИСТИННОГО автора из post_author_data.author
   │   (НЕ signer_id — это только визуальная подпись)
   │
7. Проверяем чёрный список
   │
8. Сохраняем новых участников в ReviewContestEntry (status='new')
```

### Этап 2: Нумерация (`processor.py`)

```
1. Получаем участников со статусом 'new'
   │
2. Проверяем лимит (если finish_condition = count/mixed)
   │
3. Для каждого нового участника:
   │
   ├─ Присваиваем номер (max_number + 1)
   │
   ├─ Формируем комментарий из шаблона: "{number}" → "42"
   │
   ├─ Подставляем глобальные переменные
   │
   ├─ Отправляем комментарий от имени сообщества (wall.createComment)
   │
   └─ Обновляем статус на 'commented'
```

### Этап 3: Завершение (`finalizer.py`)

```
1. Проверяем условия завершения
   │
   ├─ mixed: Если участников < target_count → {skipped: true}
   │
   ├─ count: Если участников < target_count → {skipped: true}
   │
   └─ date: Розыгрыш состоится при любом количестве > 0
   │
2. Очищаем просроченные баны в ЧС
   │
3. Фильтруем участников по активному ЧС
   │
4. Случайно выбираем победителя
   │
5. Получаем свободный промокод
   │
6. Отправляем ЛС победителю (messages.send)
   │
   └─ Fallback: Если ЛС не отправлено → комментарий под постом с template_error_comment
   │
7. Генерируем изображение-доказательство (если use_proof_image=true)
   │
   └─ create_random_proof_image() → upload_wall_photo()
   │
8. Публикуем пост с итогами (wall.post)
   │
9. Добавляем победителя в ЧС (если auto_blacklist=true)
   │
10. Обновляем статусы: winner='winner', остальные='used'
   │
11. Создаем запись в журнале доставки (ReviewContestDeliveryLog)
```

### После завершения (Post Tracker)

```python
_create_next_cyclic_post(db, post)  # Создаёт следующий триггер (через неделю)
crud.delete_system_post(db, post.id)  # Удаляет текущий триггер
```

## 🎨 Изображение-доказательство розыгрыша

При включённом `use_proof_image` генерируется изображение с:
- Номером победителя
- Именем и аватаром победителя
- Общим количеством участников
- Мини-аватарками участников (до 25 шт)
- Названием группы и конкурса
- Датой и временем розыгрыша

Генератор: `utils/random_proof_image.py` → `create_random_proof_image()`

## ⚠️ Важные моменты

1. **Один конкурс на проект** — `project_id` уникален в таблице
2. **Комментарии от сообщества** — требуется `communityToken` в настройках проекта
3. **Определение автора** — используется `post_author_data.author` (требует админский токен)
4. **Права на сообщения** — для отправки ЛС победителю нужен доступ к messages
5. **Проверка чёрного списка** — участники в бане не добавляются и не выигрывают
6. **Автоматический цикл** — триггерится Post Tracker через системный пост `contest_winner`

## 🔗 Связанные компоненты

- **Global Variables**: подстановка переменных в шаблоны
- **VK Service**: комментарии, сообщения, посты
- **SystemListPost**: кеш постов для поиска
- **Post Tracker**: автоматический запуск розыгрыша
- **random_proof_image**: генерация изображения-доказательства
- **administered_groups**: поиск админского токена
