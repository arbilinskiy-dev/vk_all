## 2. Ключевые хуки

### useConversations

Загружает список диалогов из списка рассылки проекта.

**Параметры:** `{ projectId, channel, filterUnread }`

**Возвращает:**
- `subscribers` — массив подписчиков
- `conversations` — массив `Conversation[]` (маппинг из подписчиков)
- `unreadCountsMap` — словарь `{ vk_user_id: unread_count }`
- `lastMessagesMap` — словарь `{ vk_user_id: VkMessageItem }`
- `fetchData()`, `loadMore()`, `refresh()` — управление загрузкой

**Стратегия unread-first:** На первой странице параллельно загружаются обычная страница и непрочитанные, затем мержатся — непрочитанные всегда первые.

### useMessageHistory

Работа с историей сообщений в активном диалоге.

**Параметры:** `{ projectId, userId, groupId }`

**Возвращает:**
- `messages` — массив `ChatMessageData[]`
- `loadInitial()` — загрузка первой страницы
- `loadMore()` — подгрузка старых (offset-пагинация)
- `loadAll()` — загрузка всех сообщений
- `sendMessage()` — отправка с оптимистичным обновлением
- `messageStats` — статистика (incoming/outgoing count)
- `isFullyLoaded`, `totalCount`, `source`

### useMessagesSSE

Подключение к SSE-потоку сообщений.

**Параметры:** `{ projectId, callbacks, managerName }`

**Колбэки:**
- `onNewMessage` — новое сообщение
- `onMessageRead` — диалог прочитан другим менеджером
- `onUserRead` — пользователь VK прочитал
- `onUserTyping` — пользователь печатает
- `onDialogFocus` — менеджер открыл/покинул диалог
- `onMailingUserUpdated` — обновление данных пользователя
- `onAllRead` — все диалоги прочитаны

**Автопереподключение:** EventSource автоматически переподключается при обрыве.

### useGlobalUnreadSSE

Глобальный SSE-стрим для счётчика непрочитанных в навигации.

### useTypingState

Управление индикаторами «печатает» и фокусами менеджеров.

- `typingUsers` — Set vk_user_id печатающих пользователей (сброс через 6с)
- `dialogFocuses` — Map: vk_user_id → список менеджеров в диалоге

## 3. Ключевые типы (types.ts)

### Модель сообщения — ChatMessageData

```typescript
interface ChatMessageData {
    id: string;
    direction: 'incoming' | 'outgoing';
    text: string;
    timestamp: string;         // ISO строка
    isRead: boolean;
    attachments?: MessageAttachment[];
    keyboard?: MessageKeyboard;
    sentByName?: string;       // Имя менеджера-отправителя
    isBotMessage?: boolean;    // Связано с ботом
}
```

### Модель диалога — Conversation

```typescript
interface Conversation {
    id: string;
    user: ConversationUser;
    lastMessage?: ChatMessageData;
    unreadCount: number;
    channel: 'vk' | 'tg';
    projectId: string;
}
```

### Модель пользователя — MailingUserInfo

Данные пользователя из таблицы рассылки, обогащённые информацией из VK API (`users.get`):
- Имя, фамилия, аватар, пол, дата рождения
- Город, страна, домен, платформа
- Даты первого и последнего сообщения
- Статус диалога, источник
- `last_incoming_message_date`, `last_outgoing_message_date`

### SSE-события

Типизированы через дискриминированные union-типы:
- `SSENewMessageData`
- `SSEMessageReadData`
- `SSEAllReadData`
- `SSEUserReadData`
- `SSEUserTypingData`
- `SSEDialogFocusData`
- `SSEMailingUserUpdatedData`

## 4. Компонентная иерархия

```
App.tsx
  ├─ ConversationsSidebar (левый сайдбар)
  │   ├─ MailingOnboarding (если нет токена/callback)
  │   ├─ Поиск по имени
  │   ├─ Фильтр «все / непрочитанные»
  │   ├─ Кнопка «Прочитать все»
  │   └─ ConversationItem[] (список диалогов)
  │       ├─ Аватар + имя
  │       ├─ Последнее сообщение (превью)
  │       └─ Бейдж непрочитанных
  │
  └─ MessagesPage (основная область)
      ├─ MessagesEmptyState (если диалог не выбран)
      │
      ├─ ChatView (активный диалог)
      │   ├─ ChatHeader
      │   │   ├─ Имя + статус пользователя
      │   │   ├─ «Печатает...» / другие менеджеры
      │   │   └─ Кнопки: обновить, VK ссылка, пометить непрочитанным
      │   │
      │   ├─ ChatMessageList
      │   │   ├─ Кнопка «Загрузить старые» / «Загрузить все»
      │   │   └─ ChatMessage[]
      │   │       ├─ Текст + время + имя менеджера
      │   │       ├─ ChatMessageAttachments (фото, файлы, аудио)
      │   │       └─ ChatMessageKeyboard (кнопки бота)
      │   │
      │   └─ ChatInput
      │       ├─ ChatInputToolbar (emoji, переменные)
      │       ├─ ChatInputVariablesBar
      │       └─ ChatInputAttachments (превью прикреплённых файлов)
      │
      └─ UserInfoPanel (правая панель)
          ├─ UserInfoPanelHeader (аватар, имя, VK ссылка)
          ├─ UserInfoProfileTab (детали профиля)
          ├─ AttachmentsTab (вложения диалога)
          └─ UserPostsTab (посты пользователя)
```
