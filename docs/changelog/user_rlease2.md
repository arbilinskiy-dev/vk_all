## 🔴 Открытые задачи

### Режим «По количеству» в конкурсе отзывов не работает автоматически

**Проблема:**  
Если в настройках конкурса отзывов выбрать условие завершения «По количеству», итоги никогда не подведутся автоматически. Конкурс будет бесконечно ждать, хотя интерфейс обещает автоматическое подведение.

**Статус:** 🔴 В работе

---

### Конкурс может показывать 0 участников без объяснений

**Проблема:**  
Если в системе не настроен токен администратора группы, конкурс отзывов не сможет определить авторов постов. Каждую неделю будет показывать «Добавлено 0 участников» без какого-либо предупреждения.

**Статус:** 🔴 В работе

## ✅ Решённые задачи

### Подборка слетала при импорте товаров из буфера

**Проблема:**  
При создании нескольких товаров через «Из буфера», если в скопированной таблице была колонка «Подборка» с текстовым названием (например, «Сеты»), подборка не применялась к товарам. После нажатия «Импортировать» все товары показывались как «Без подборки». Также не было возможности сопоставить колонку с ссылкой на фото.

**Решение:**  
Теперь система автоматически создаёт недостающие подборки. Если вы указываете подборку «Сеты», а такой подборки ещё нет — она будет создана автоматически, и все товары сразу попадут в неё. Добавлена опция «Фото (URL)» для маппинга ссылки на фото из таблицы.

---

### Исправлена ошибка при сборе участников конкурса отзывов

**Проблема:**  
При нажатии «Собрать отзывы» возникала внутренняя ошибка сервера (500). Участники не добавлялись.

**Решение:**  
Исправлена проблема с внутренним конфликтом сессий базы данных. Теперь сбор участников работает стабильно — отзывы корректно находятся, авторы определяются, записи создаются.

---

### Улучшена надёжность автоматического цикла конкурса

**Проблема:**  
Автоматический еженедельный цикл конкурса мог остановиться из-за различных внутренних ошибок, и пользователь не всегда получал уведомления о проблемах.

**Решение:**  
Внесено 14 исправлений в автоматический цикл конкурса. Теперь:
- Ошибки на одном этапе не останавливают весь цикл — следующая неделя будет создана в любом случае.
- Если что-то пошло не так — в календаре автоматически появляется заметка с описанием проблемы.
- Исправлено определение часовых поясов для точного срабатывания по расписанию.

## ✅ Решённые задачи

### Плавная загрузка фотографий в разделе «Товары»

**Проблема:**  
Фотографии товаров загружались рывками — появлялись в случайном порядке по мере загрузки с сервера ВКонтакте. Это выглядело непрофессионально, особенно при большом количестве товаров.

**Решение:**  
Теперь на месте каждой фотографии сначала отображается серый пульсирующий плейсхолдер. Как только фото загрузилось, оно плавно проявляется. Текстовая информация (название, описание, цена) отображается сразу, не дожидаясь загрузки изображений.

---

### Улучшена адаптивность страницы конкурса отзывов

**Проблема:**  
При уменьшении размера окна (или на небольших экранах) элементы на странице настроек конкурса отзывов отображались некорректно: тексты на кнопках обрезались, тумблер-переключатели выглядели увеличенными, а панель предпросмотра вытесняла настройки.

**Решение:**  
- Кнопки выбора условий подведения итогов теперь автоматически переносятся на следующую строку при нехватке ширины.
- Кнопки вставки переменных в шаблонах сообщений больше не выходят за границы.
- Тумблер-переключатели теперь имеют корректный размер и не деформируются.
- Настройки получили приоритет по ширине — при сжатии окна превью уменьшается первым.

### Редактирование и удаление подборок товаров

**Что добавили:**

**Что теперь можно:**  
Управлять подборками полностью внутри приложения: создавать, переименовывать и удалять. Все изменения синхронизируются с ВКонтакте.

## ✅ Решённые задачи

### Пустые подборки больше не пропадают

**Проблема:**  
Если создать подборку и не добавить в неё товары, после обновления страницы она исчезала.

**Решение:**  
Теперь пустые подборки сохраняются в приложении даже после обновления данных. Вы можете создать подборку заранее и наполнить её позже.

---

### Устранены лишние ошибки при работе с подборками

**Проблема:**  
Операции с подборками (создание, редактирование, удаление) иногда выполнялись медленнее из-за нескольких неудачных попыток подключения.

**Решение:**  
Оптимизирован выбор ключа доступа для операций с товарами. Теперь система сразу использует правильный ключ администратора группы, что ускоряет выполнение операций.

---

### Защита от случайного двойного создания подборки

**Проблема:**  
Если случайно нажать кнопку «Ок» дважды при создании подборки, создавались две одинаковые подборки.

**Решение:**  
Теперь кнопки создания и сохранения блокируются на время выполнения операции. При повторном клике ничего не произойдёт — подборка создастся только один раз.

## ✅ Решённые задачи

### Товары терялись при массовом создании из файла/буфера

**Проблема:**  
При массовом создании товаров через CSV или буфер обмена все товары не создавались — ВКонтакте отклонял фотографии с ошибкой о неправильном размере изображения.

**Решение:**  
Система теперь сохраняет и использует полноразмерные фотографии товаров вместо маленьких превью. Фотографии корректно загружаются в ВК, товары создаются без ошибок.

---

### Много ошибок доступа при создании товаров

**Проблема:**  
При создании товаров в логах появлялось по 5 ошибок «Доступ запрещён» на каждый товар, хотя в итоге товар создавался. Процесс занимал в 3-5 раз больше времени.

**Решение:**  
Оптимизирован выбор ключа доступа для создания товаров. Теперь система сразу использует ключ администратора группы, минуя нерабочие ключи. Создание 12 товаров ускорилось с ~10 минут до ~2 минут.

---

### Фото товара иногда не загружалось из-за временной ошибки сервера

**Проблема:**  
При массовом создании один или несколько товаров могли не создаться, если сервер ВКонтакте временно не мог отдать фотографию.

**Решение:**  
Добавлена автоматическая повторная попытка загрузки фото. Если сервер временно недоступен, система пробует ещё 2 раза с короткой паузой. Если фото слишком большое — автоматически подбирает меньший подходящий размер.

---

### Подборка создавалась 3 раза вместо одной

**Проблема:**  
При массовом создании товаров, если товары добавлялись в новую подборку, система могла создать 2-3 одинаковые подборки вместо одной.

**Решение:**  
Исправлена внутренняя логика повторных попыток. Теперь операции создания (подборки, товары) не дублируются при медленном ответе сервера.

---

### Подготовка к стабильной работе на сервере

**Проблема:**  
При длительной работе на сервере соединения с базой данных могли «протухать», что приводило к случайным ошибкам.

**Решение:**  
Проведён аудит и оптимизация работы с базой данных. Все соединения теперь корректно открываются и закрываются, что обеспечивает стабильную работу при длительном использовании.

## 🆕 Новые возможности

### Прогресс-бар при массовом создании товаров

**Что добавили:**  
При массовом создании товаров теперь отображается полоса прогресса с подробной информацией о ходе выполнения.

**Как использовать:**  
1. Перейдите в раздел «Товары».
2. Загрузите товары из CSV или буфера обмена.
3. Нажмите «Создать».
4. В окне появится полоса прогресса: сколько товаров создано из общего числа, название текущего товара, количество успешных и неудачных.
5. Кнопка «Создать» показывает текущий прогресс (например, «3/12»).

**Что теперь можно:**  
Видеть в реальном времени, как идёт процесс создания. Понимать, какие товары создались успешно, а какие — с ошибкой, не дожидаясь окончания всего процесса.

---

### Удаление и редактирование товаров работает значительно быстрее

**Проблема:**  
При удалении или редактировании товаров система долго перебирала неподходящие аккаунты, из-за чего операция выполнялась медленно. В логах появлялись многочисленные ошибки «VK API Error 15».

**Решение:**  
Теперь при работе с товарами (удаление, редактирование, перемещение между подборками) система сразу определяет, кто является администратором группы, и использует именно его аккаунт. Лишних попыток больше нет.

---

### Массовое сохранение нескольких товаров могло завершиться ошибкой

**Проблема:**  
При одновременном сохранении 2 и более товаров, если первый товар не удалось обновить (например, из-за временной ошибки ВКонтакте), все остальные товары тоже не сохранялись — отображалось «Успешно: 0 из 2, Ошибок: 2».

**Решение:**  
Теперь ошибка одного товара не влияет на остальные. Если из 5 товаров 1 не удалось обновить — остальные 4 будут успешно сохранены. Неудачный товар останется подсвеченным для повторной попытки.

---

### После массового удаления товаров интерфейс не обновлялся

**Проблема:**  
После массового удаления товаров (выделить несколько → удалить) удалённые карточки продолжали отображаться в списке. Счётчик товаров в подборке показывал старое число. Приходилось перезагружать страницу вручную.

**Решение:**  
Теперь после удаления: карточки сразу исчезают из списка, счётчик подборки мгновенно пересчитывается, выделение сбрасывается. Никаких ручных действий не требуется.

---

### Префикс «NEW» не добавлялся при импорте товаров из файла

**Проблема:**  
При загрузке товаров из файла (CSV/Excel) с включённой галочкой «Добавить NEW к названию» префикс не применялся. Все товары появлялись в таблице с оригинальными названиями.

**Решение:**  
Теперь галочка «NEW» работает корректно — при включённом флаге все импортированные названия товаров автоматически получают префикс «NEW» в начале.

---

### Лишние поля при импорте новых товаров из файла

**Проблема:**  
При импорте товаров из файла для **создания** новых товаров в окне сопоставления колонок отображались поля «VK ID товара» и «VK ссылка на товар». Эти поля не нужны при создании и могли запутать.

**Решение:**  
Теперь при создании новых товаров эти поля скрыты. Они отображаются только в режиме обновления существующих товаров, где действительно необходимы.

---

### Ошибки валидации при массовом создании были незаметны

**Проблема:**  
При нажатии «Создать товары», если в таблице были незаполненные обязательные поля, ничего не происходило — ни уведомления, ни подсказки. Приходилось самостоятельно искать проблемные строки.

**Решение:**  
Теперь при ошибках валидации:
- Появляется уведомление с указанием количества строк с ошибками.
- Таблица автоматически прокручивается к первой проблемной строке, чтобы сразу было видно, что нужно исправить.

## 🆕 Новые возможности

### Отмена массового создания товаров

**Что добавили:**  
Теперь процесс массового создания товаров можно остановить в любой момент. Прогресс-бар перенесён вниз рядом с кнопками управления и всегда виден.

**Как использовать:**  
1. Начните массовое создание товаров (загрузите из файла/буфера → нажмите «Создать»).

4. Подтвердите остановку — уже созданные товары сохранятся, остальные не будут созданы.

**Что теперь можно:**  
Остановить процесс, если заметили ошибку в данных или передумали. Не нужно ждать создания всех товаров — уже созданные останутся на месте.

## ✅ Решённые задачи

### Группировка записей по разделам в Центре обновлений

**Проблема:**  
В «Центре обновлений» все записи отображались простым списком внутри каждой категории. При большом количестве изменений было сложно быстро найти обновления, относящиеся к конкретному разделу — например, только к товарам или только к конкурсу отзывов.

**Решение:**  
Теперь записи внутри каждой категории (Новые возможности / Улучшения / Исправления) автоматически группируются по разделу приложения. Каждая группа имеет свой подзаголовок с названием раздела и количеством изменений — так сразу видно, сколько обновлений затронуло конкретную область.

### Текст с переносами строк ломал импорт из буфера обмена

**Проблема:**  
При копировании таблицы из Google Sheets или Excel, если ячейка содержала текст с переносами строк (Enter), данные при вставке «сползали» — строки путались, колонки не совпадали, в таблице появлялись лишние пустые строки.

**Решение:**  
Переработан алгоритм разбора данных из буфера обмена. Теперь ячейки с многострочным текстом корректно распознаются и не ломают структуру таблицы. Текст внутри ячейки сохраняется полностью, включая переносы строк.

---

### Категория не применялась при импорте товаров из буфера

**Проблема:**  
При массовом создании товаров через «Из буфера», если в скопированной таблице была колонка «Категория», категория не применялась к товарам. В таблице создания все товары отображались без категории, хотя данные были корректными.

**Решение:**  
Исправлено распознавание категорий. Теперь система корректно понимает формат «Название категории (ID)» из скопированных данных и автоматически находит нужную категорию.

---

### Данные не обновлялись после создания товаров

**Проблема:**  
После создания товаров (одного или нескольких) интерфейс не обновлялся: счётчики товаров в подборках оставались прежними, новые товары не появлялись в списке. Приходилось обновлять страницу вручную.

**Решение:**  
Теперь после создания товаров данные автоматически синхронизируются с ВКонтакте. Список товаров обновляется, счётчики подборок пересчитываются — всё происходит без ручного обновления.

---

### Кириллические символы отображались как «кракозябры» при импорте CSV

**Проблема:**  
При экспорте товаров в CSV, редактировании в Google Sheets и повторном импорте — вместо русского текста отображались нечитаемые символы. Названия товаров, описания и заголовки колонок были полностью нечитаемы.

**Решение:**  
Полностью переработан импорт CSV-файлов. Теперь система автоматически определяет формат файла (CSV или Excel) и использует правильный способ чтения. CSV-файлы читаются с корректной поддержкой русского языка, включая файлы из Google Sheets, Excel и других редакторов.

---

### Автоматическое определение колонок при импорте из буфера

**Что добавили:**  
Три улучшения модального окна «Импорт данных из буфера» в разделе товаров.

**Как использовать:**  
1. Откройте раздел «Товары» → нажмите «Из буфера».
2. Скопируйте таблицу из Excel/Google Sheets (с заголовками «Название», «Описание», «Цена» и т.д.) и вставьте в поле.
3. Система автоматически определит колонки по заголовкам и выставит нужные поля.

**Что теперь можно:**  
- Колонки определяются автоматически — не нужно вручную выставлять каждое поле.
- Одно поле нельзя выбрать дважды — занятые поля блокируются с пометкой «занято».
- Выпадающие списки стали красивее и удобнее — кастомный дизайн вместо стандартного браузерного.

### Посты теперь публикуются мгновенно без задержки на проверку

**Проблема:**  
После публикации системного поста на карточке появлялась вращающаяся шестерёнка — система ждала до 50 секунд, проверяя, действительно ли пост вышел на стене ВКонтакте. Если за 5 минут подтверждение не приходило, карточка показывала предупреждение «Возможная ошибка», хотя пост на самом деле уже находился на стене.

**Решение:**  
Убрана лишняя проверка. Теперь если ВКонтакте подтвердил публикацию (вернул ID поста) — пост считается опубликованным сразу. Шестерёнка и предупреждение «Возможная ошибка» больше не появляются. Следующий пост в цикле (для автоматизаций) создаётся мгновенно, без задержки.

**Что добавили:**  
Новая кнопка «Отредактировать все» в разделе «Предложенные посты». Позволяет отправить все отзывы на AI-коррекцию одним нажатием, вместо того чтобы нажимать «Редактор AI» для каждого поста по отдельности.

**Как использовать:**  
1. Откройте раздел «Предложенные» в левом меню.
2. Выберите проект, у которого есть предложенные посты.
3. Рядом с кнопкой «Обновить» появится кнопка **«Отредактировать все (N)»**, где N — количество постов.
4. Нажмите на неё — все посты будут отправлены на AI-обработку одним запросом.
5. После обработки каждый пост отобразится в режиме «пост слева — исправленный текст справа».
6. Исправленный текст можно отредактировать вручную и скопировать кнопкой «Копировать».

**Что теперь можно:**  
- Обработать все отзывы за один клик вместо 10–20 отдельных нажатий.
- AI исправляет ошибки, добавляет хештег и благодарность для всех постов сразу.
- Экономия времени и запросов к нейронке — все посты обрабатываются в одном запросе.

### Исправлена ошибка загрузки данных на предпроде (502 Bad Gateway)

**Проблема:**  
При открытии приложения первая порция данных не загружалась — выскакивала ошибка 502. Также при нажатии кнопки обновления проектов в боковой панели происходила аналогичная ошибка, из-за чего данные не обновлялись.

**Решение:**  
Теперь система автоматически повторяет запрос до 3 раз с нарастающей паузой, если сервер временно недоступен. Кроме того, обновление проектов теперь загружает данные небольшими порциями, а не все сразу — это снижает нагрузку на сервер и предотвращает ошибки.

### Добавлена иконка во вкладку браузера

**Как использовать:**

**Что теперь можно:**
