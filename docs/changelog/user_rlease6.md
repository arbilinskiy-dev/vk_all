# 📋 Релиз 6 — Стабильность модуля сообщений

---

## ✅ Решённые задачи

### Сообщения не всегда доходят до менеджера в реальном времени

**Проблема:**  
При получении новых сообщений от пользователей ВКонтакте уведомления в чате работали нестабильно. Иногда новое сообщение появлялось мгновенно, а иногда — не приходило вовсе, требуя ручного обновления страницы. Особенно часто проблема проявлялась при первом открытии модуля сообщений или после временной потери интернет-соединения.

**Решение:**  
Переработана система доставки уведомлений. Теперь система корректно инициализируется при запуске сервера, а не при первом подключении пользователя. При высокой нагрузке (много сообщений одновременно) старые уведомления автоматически вытесняются новыми вместо молчаливой потери. После обрыва соединения клиент получает пропущенные события о прочтении диалогов.

---

### Отправленные сообщения не видны другим менеджерам

**Проблема:**  
Когда один менеджер отправлял сообщение пользователю, другие менеджеры не видели это сообщение у себя в чате. Оно появлялось с задержкой от нескольких секунд до нескольких минут, пока ВКонтакте не присылало подтверждение обратно на сервер.

**Решение:**  
Теперь при отправке сообщения уведомление мгновенно рассылается всем подключённым менеджерам. Исходящее сообщение появляется в чате у всех без ожидания ответа от ВКонтакте. Дублирование при этом исключено.

---

### Фантомные бейджи непрочитанных после переподключения

**Проблема:**  
После кратковременной потери соединения (например, при переключении Wi-Fi) счётчики непрочитанных сообщений на диалогах могли показывать устаревшие значения. Диалог, который другой менеджер уже прочитал, продолжал отображаться как непрочитанный.

**Решение:**  
Система кеширования событий теперь сохраняет не только новые сообщения, но и события о прочтении диалогов. При переподключении клиент получает актуальные статусы прочитанности для каждого диалога.

## 🆕 Новые возможности

### Просмотр текущих настроек Callback-сервера

**Что добавили:**  
Теперь при открытии раздела «Интеграции» в настройках проекта автоматически подгружаются текущие настройки Callback-сервера из ВКонтакте. Вы можете увидеть, какой сервер настроен, его статус и на какие события он подписан.

**Как использовать:**  
1. Откройте настройки проекта
2. Раскройте раздел «Интеграции»
3. Автоматически появится информация о текущем сервере и чекбоксы событий будут проставлены по реальным настройкам
4. Для принудительного обновления нажмите «Загрузить текущие настройки из VK»

**Что теперь можно:**  
Видеть реальное состояние callback-сервера прямо в нашем интерфейсе, не заходя в настройки ВКонтакте. При повторной настройке чекбоксы соответствуют тому, что реально включено в VK, а не стартуют с «все выбраны».

---

### Ссылки на настройки Callback API в VK

**Что добавили:**  
Теперь в интерфейсе есть прямые ссылки для перехода в настройки Callback API сообщества в ВКонтакте. Не нужно искать нужную страницу вручную — один клик открывает настройки нужной группы.

**Как использовать:**  
1. В настройках проекта («Интеграции») — ссылка находится под инструкцией автонастройки
2. При массовой настройке Callback-серверов — если возникает ошибка лимита серверов, ссылка ведёт прямо в настройки нужной группы

**Что теперь можно:**  
Быстро перейти в настройки API конкретной группы ВК для ручного управления Callback-серверами (например, удалить лишние серверы при лимите).

### Карточка «В рассылке» показывала 0 при новых сообщениях

**Проблема:**  
Когда пользователь писал сообществу и автоматически добавлялся в список рассылки, карточка «В рассылке» продолжала показывать 0. При этом в таблице рассылки пользователь был виден. Счётчик обновлялся только при полной синхронизации.

**Решение:**  
Теперь при каждом новом сообщении от пользователя счётчик на карточке обновляется моментально — не нужно ждать полной синхронизации. Счётчик обновляется даже если вы находитесь на другой вкладке (например, «Подписчики»).

---

### Кнопка «Собрать все сообщения» прямо в чате

**Что добавили:**  
Теперь в заголовке списка диалогов есть кнопка для быстрого сбора всех сообщений. Раньше для этого нужно было переходить в раздел «Списки» и запускать сбор оттуда.

**Как использовать:**  
1. Откройте модуль «Сообщения» и выберите проект
2. В заголовке «Диалоги» нажмите кнопку ↓ (справа от счётчика)
3. Дождитесь завершения сбора — прогресс отображается прямо под заголовком
4. После завершения список диалогов обновится автоматически

**Что теперь можно:**  
Обновлять список подписчиков рассылки не выходя из чата. Удобно, если нужно быстро подгрузить новых пользователей, написавших в сообщество.

---

### Массовая настройка Callback не работала с первого раза

**Проблема:**  
При нажатии кнопки «Авто-настройка колбэков» (массовой или индивидуальной) всё выглядело успешно — в логах отображался «confirm», но реально callback-сервер не начинал принимать события. Сообщения, лайки и другие события не приходили. Чтобы починить, приходилось заходить в настройки каждого проекта и перенастраивать колбэк повторно.

**Решение:**  
Исправлена ошибка с порядком сохранения кода подтверждения. Теперь код сохраняется в систему до начала настройки сервера, а не после. Callback-сервер подтверждается с первого раза — повторная ручная настройка больше не требуется.

### Расширенные фильтры периодов в мониторинге сообщений

**Что добавили:**  
Фильтры периодов в разделе «Мониторинг сообщений» стали удобнее. Теперь доступны: Сегодня, Вчера, Эта неделя, Прошлая неделя, Этот месяц, Прошлый месяц, Всё время и Свой период.

**Что изменилось:**  
Старые фильтры «За неделю» и «За месяц» (которые считали за последние 7/30 дней) заменены на более точные: «Эта неделя» — с понедельника по сегодня, «Прошлая неделя» — полная прошлая неделя, «Этот месяц» — с 1-го числа, «Прошлый месяц» — полностью предыдущий месяц.

---

### Полная переработка дашборда «Мониторинг сообщений»

**Что добавили:**  
Страница мониторинга полностью переработана. Теперь вместо простых счётчиков «Входящие / Исходящие» доступна детальная аналитика: какие сообщения пришли по кнопкам, а какие — живые текстовые; какие исходящие отправлены администраторами, а какие — ботом. Добавлена таблица активности администраторов с возможностью раскрыть конкретные диалоги.

**Как использовать:**  
1. Перейдите в раздел «Мониторинг» в боковом меню
2. В сводке вверху — общие показатели + блоки «Входящие» и «Исходящие» с детальной разбивкой
3. Под графиком — таблица «Активность администраторов»: кто из менеджеров сколько сообщений отправил, сколько уникальных диалогов вёл
4. Нажмите на строку администратора — раскроется список конкретных диалогов (с фото, именем пользователя, проектом и ссылками)
5. В таблице проектов — расширенные колонки с подстроками: «X жив. / Y кн.» для входящих, «X адм. / Y бот» для исходящих, плюс отдельная колонка «Диалоги»

**Что теперь можно:**  
- Видеть, сколько обращений пришло «живых» (текстовых), а сколько — автоматических (по кнопкам)
- Отслеживать нагрузку каждого администратора: кто сколько сообщений отправил и скольким пользователям написал
- Просматривать конкретные диалоги администратора с переходом в VK
- Понимать, какую долю ответов генерирует бот, а какую — живые менеджеры
- Видеть в таблице проектов полную разбивку по типам для каждого сообщества

### Скрытие сообщений бота/рассылки в чате

**Что добавили:**  
В панели «Фильтры отображения» появился новый тумблер «Скрыть бот/рассылку». При включении из переписки убираются все автоматические сообщения с метками «Бот / Рассылка» и «Кнопка» — остаётся только живая переписка с пользователем.

**Как использовать:**  
1. Откройте любой диалог в модуле «Сообщения»
2. Нажмите кнопку-воронку «Фильтры отображения» в шапке чата
3. Включите тумблер «Скрыть бот/рассылку»
4. Сообщения с жёлтыми метками исчезнут из переписки

**Что теперь можно:**  
Быстро отфильтровать автоматические сообщения бота и рассылки, чтобы сосредоточиться на живом общении с пользователем.

---

### Автоматическая пометка «Прочитано» при ответе пользователя

**Что добавили:**  
Если пользователь прислал вам ответное сообщение — все ваши предыдущие исходящие сообщения автоматически помечаются как прочитанные (зелёный кружок). Раньше это зависело только от события VK «пользователь прочитал», которое приходило не всегда.

**Как использовать:**  
Ничего специального делать не нужно — работает автоматически. При открытии диалога или получении нового сообщения от пользователя статусы обновляются сами.

**Что теперь можно:**  
Надёжнее видеть, какие из ваших сообщений пользователь уже прочитал, даже если VK не прислал уведомление о прочтении.
