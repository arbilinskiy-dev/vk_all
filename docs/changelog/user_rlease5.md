## 🆕 Новые возможности

### Отображение кнопок VK в сообщениях

**Что добавили:**  
Теперь в чате с пользователем видны кнопки, которые были прикреплены к сообщениям ботом или сообществом ВКонтакте (inline-клавиатура). Раньше кнопки были скрыты и менеджер не видел, какие действия были доступны пользователю.

**Как выглядит:**  
Кнопки отображаются внутри пузыря сообщения между текстом и временем отправки. Цвета соответствуют VK: синие (основные), зелёные (подтверждение), красные (отмена), белые (второстепенные). Кнопки-ссылки открывают URL по клику.

**Что теперь можно:**  
Менеджер видит полную картину диалога — какие кнопки были у пользователя, на что он мог нажать. Это помогает понять контекст разговора и действия бота.

## 🔧 Улучшения

### Ускорена загрузка истории сообщений

**Что улучшили:**  
При открытии диалога или нажатии «Загрузить все сообщения» данные теперь сохраняются в базу значительно быстрее — одной операцией вместо сотен отдельных.

**Что изменилось:**  
Загрузка 200 сообщений стала быстрее в десятки раз. Особенно заметно при полной загрузке истории (тысячи сообщений) — вместо 10-30 секунд ожидания, операция выполняется за 1-2 секунды.

---

### Входящие сообщения появляются без ручного обновления

**Что улучшили:**  
Раньше, когда пользователь писал сообщество, новое сообщение не появлялось в чате менеджера — нужно было вручную нажимать «Обновить» или переключаться между диалогами.

**Что изменилось:**  
Теперь входящие сообщения от пользователей автоматически сохраняются в базу в момент получения. При следующем обращении к диалогу менеджер сразу увидит новые сообщения без необходимости обновлять страницу.

---

### Открытие диалога стало быстрее

**Что улучшили:**  
При открытии диалога выполнялось два отдельных запроса к серверу: один за историей сообщений, второй за информацией о пользователе (имя, фото, даты сообщений). Это замедляло загрузку.

**Что изменилось:**  
Теперь при первой загрузке истории диалога информация о пользователе подгружается в том же запросе. Диалог открывается быстрее, лишний запрос не выполняется.

## 🔧 Улучшения

### Исходящие сообщения от чат-ботов больше не создают непрочитанных диалогов

**Что улучшили:**  
Когда к VK-группе подключены сторонние чат-боты (которые автоматически отправляют сообщения от имени группы), каждое такое сообщение создавало "непрочитанный диалог" для менеджера. В результате в списке диалогов появлялись мусорные записи, которые менеджер не инициировал и не ожидал.

**Что изменилось:**  
Теперь флаг «непрочитанное сообщение» появляется только когда реальный пользователь пишет в группу. Исходящие сообщения (от чат-ботов, автоответчиков и других систем) по-прежнему видны в чате, но не увеличивают счётчик непрочитанных и не создают лишних диалогов.

## ✅ Решённые задачи

### Не сбрасывался счётчик непрочитанных сообщений

**Проблема:**  
После входа в диалог и прочтения сообщений счётчик непрочитанных (badge) не обнулялся. При перезагрузке проекта диалог по-прежнему показывал «2 непрочитанных», хотя все сообщения были прочитаны.

**Решение:**  
Теперь при открытии диалога счётчик непрочитанных сразу сбрасывается в ноль, не дожидаясь ответа от сервера. Также исправлена обработка событий об обнулении счётчика — теперь badge корректно обновляется в реальном времени.

---

### В конкурсе отзывов показывалась неправильная дата

**Проблема:**  
В таблице постов конкурса отзывов все посты показывали одинаковую дату — дату нажатия кнопки «Собрать посты», а не реальную дату публикации поста ВКонтакте.

**Решение:**  
Теперь в колонке «Дата» отображается настоящая дата публикации поста ВКонтакте. Для ранее собранных постов будет показываться прежняя дата сбора — при повторном сборе данные обновятся.

---

### В конкурсе отзывов неверно отображалось «участники» вместо «посты»

**Проблема:**  
Интерфейс показывал «Найдено участников: 6», хотя на самом деле один человек мог оставить несколько отзывов. Это вводило в заблуждение — казалось, что один автор = одна запись, хотя система нумерует каждый пост отдельно.

**Решение:**  
Вся терминология обновлена: теперь интерфейс показывает «Найдено постов: 6» и отдельно «Уникальных авторов: N». При подведении итогов система выбирает «пост-победитель», а приз отправляется автору этого поста. Все тексты кнопок, модальных окон и уведомлений приведены в соответствие с реальной логикой.

## ✅ Решённые задачи (продолжение)

### Массовый сбор подписчиков не работал — все воркеры падали

**Проблема:**  
При запуске сбора подписчиков из админ-панели процесс завершался с результатом 0 из 206 проектов. Все рабочие процессы (воркеры) падали сразу после запуска.

**Решение:**  
Исправлена ошибка в коде воркера, из-за которой он не мог прочитать название проекта. Теперь сбор подписчиков работает корректно — все токены обрабатывают проекты параллельно.

---

### Новый пользователь не появлялся в списке диалогов

**Проблема:**  
Когда новый пользователь (которого ещё не было в списке диалогов) писал сообщение в сообщество, его диалог не появлялся в списке слева. Диалог становился доступен только после перезагрузки страницы.

**Решение:**  
Теперь когда новый пользователь пишет первое сообщение в сообщество, его диалог мгновенно появляется вверху списка диалогов в реальном времени — как в VK или Telegram. Не нужно обновлять страницу.

---

### Callback-сервер VK не доставлял события (устаревший URL)

**Проблема:**  
В одном из проектов («тестовая группа 122») callback-события от VK перестали приходить. Сообщения от пользователей не появлялись в интерфейсе вообще, хотя в других проектах всё работало.

**Решение:**  
Callback-сервер был настроен на старый адрес (ngrok URL), который стал недоступен после перезапуска. URL обновлён. Важно: при каждом перезапуске ngrok нужно нажимать «Настроить автоматически» в настройках каждого проекта, чтобы обновить URL.

---

### Улучшено логирование ошибок Callback API

**Проблема:**  
При ошибках разбора callback-событий от VK в логи записывалось только «Не удалось распарсить JSON» — без деталей о том, от какой группы пришёл запрос и почему не распарсился.

**Решение:**  
Теперь в логи записывается IP-адрес, тип содержимого, тело запроса и текст ошибки. Это помогает быстро находить проблемы с доставкой callback-событий.

## 🆕 Новые возможности

### Автонастройка Callback-сервера VK

**Что добавили:**  
В настройках проекта (раздел «Интеграции») появилась кнопка «Настроить автоматически», которая в один клик создаёт и настраивает Callback-сервер ВКонтакте для приёма уведомлений о событиях в группе.

**Как использовать:**  
1. Откройте настройки проекта → раздел «Интеграции»
2. Убедитесь, что указан токен сообщества
3. Нажмите кнопку «Настроить автоматически»
4. Дождитесь зелёного уведомления об успехе

**Что теперь можно:**  
Больше не нужно заходить в настройки группы ВКонтакте и вручную вводить URL сервера, код подтверждения, выбирать галочки событий. Всё происходит автоматически. При работе на локалке система сама определяет ngrok-туннель и настраивает callback через него.

### Массовая настройка Callback-серверов VK одной кнопкой

**Что добавили:**  
На странице «Управление базой проектов» появилась кнопка «Настроить колбэки». Она автоматически настраивает Callback-серверы ВКонтакте для всех проектов, у которых указан токен сообщества. Раньше нужно было заходить в настройки каждого проекта по отдельности.

**Как использовать:**  
1. Откройте «Управление базой проектов»
2. Нажмите кнопку «Настроить колбэки» (фиолетовая, с иконкой молнии) в тулбаре
3. В открывшемся окне увидите количество проектов-кандидатов и текущее окружение (локалка/сервер)
4. Нажмите «Настроить все колбэки» и наблюдайте за прогрессом
5. По завершении увидите сводку: сколько серверов создано, обновлено и где были ошибки

**Что теперь можно:**  
Одной кнопкой настроить Callback-серверы для 200+ проектов за пару минут. Можно остановить процесс в любой момент. Все ошибки (например, лимит 10 серверов в VK) отображаются с подсказками.

### Количество непрочитанных диалогов в сайдбаре проектов

**Что добавили:**  
Когда вы находитесь в модуле «Сообщения ВКонтакте», в списке проектов слева теперь рядом с каждым проектом отображается бейдж — количество диалогов, в которых есть непрочитанные сообщения. Считаются именно чаты (диалоги), а не общее количество сообщений.

**Как использовать:**  
1. Перейдите в модуль «Сообщения ВКонтакте» через левое меню
2. В сайдбаре проектов у каждого проекта появится число — сколько диалогов с непрочитанными
3. Если непрочитанных нет — бейдж не отображается
4. При чтении сообщений в открытом проекте счётчик обновляется в реальном времени

**Что теперь можно:**  
Быстро видеть, в каких проектах есть непрочитанные сообщения, не заходя в каждый проект отдельно. Удобно для управления несколькими сообществами одновременно.

### Понятное сообщение при лимите Callback-серверов

**Что добавили:**  
Раньше при попытке автонастройки Callback-сервера, если в сообществе уже было 10 серверов (максимум VK), выводилась непонятная техническая ошибка. Теперь отображается понятное предупреждение с объяснением проблемы и прямой ссылкой на страницу управления API сообщества.

**Как использовать:**  
1. Если при автонастройке вы увидите жёлтое предупреждение «Достигнут лимит Callback-серверов» (или «Лимит Callback-серверов» в сообщениях) — нажмите ссылку «Открыть настройки API сообщества»
2. В открывшейся странице VK удалите неиспользуемые серверы (со статусом «Ошибка» — в первую очередь)
3. Вернитесь и повторите автонастройку

**Что теперь можно:**  
Быстро понять, что именно мешает настройке, и сразу перейти на нужную страницу VK для решения проблемы — без поиска инструкций.

### Непрочитанные диалоги автоматически в топе

**Что добавили:**  
При открытии проекта в модуле «Сообщения» диалоги с непрочитанными сообщениями теперь автоматически отображаются в самом верху списка. Не нужно прокручивать весь список, чтобы найти новые сообщения.

**Как использовать:**  
1. Откройте любой проект в модуле «Сообщения»
2. Диалоги с непрочитанными сообщениями уже будут вверху списка
3. Чем больше непрочитанных — тем выше диалог
4. Прочитанные диалоги располагаются ниже по дате последнего сообщения

**Что теперь можно:**  
Сразу видеть и обрабатывать диалоги, требующие ответа, без ручного поиска по списку. Особенно полезно для сообществ с большим количеством диалогов.

### Счётчики непрочитанных обновляются в реальном времени для всех проектов

**Что добавили:**  
Раньше счётчик непрочитанных диалогов рядом с названием проекта обновлялся только при входе в модуль «Сообщения» и только для активного проекта. Если кто-то писал в другой проект — бейдж не появлялся. Если другой менеджер прочитал диалог — ваш бейдж не уменьшался.

Теперь счётчики обновляются мгновенно для ВСЕХ проектов:
- Новое сообщение в любой проект → бейдж тут же увеличивается
- Менеджер прочитал диалог → бейдж уменьшается у всех

**Как использовать:**  
1. Откройте модуль «Сообщения» (боковое меню)
2. Счётчики рядом с каждым проектом обновляются автоматически
3. Работает даже когда вы находитесь внутри другого проекта
4. При работе нескольких менеджеров — все видят актуальные счётчики

**Что теперь можно:**  
Видеть актуальное количество необработанных диалогов по каждому проекту без перезагрузки страницы. Координировать работу команды — когда один менеджер обработал диалог, у остальных счётчик снижается автоматически.

### Фильтры «Подключено / Не подключено» и «Есть новые» в сайдбаре проектов

**Что добавили:**  
Когда вы находитесь в модуле «Сообщения», в списке проектов слева теперь есть два дополнительных фильтра:
- **Callback API** — показать только проекты, у которых подключён callback-сервер (или наоборот — только где не подключён).
- **Диалоги** — показать только проекты, где есть непрочитанные сообщения.

**Как использовать:**  
1. Перейдите в модуль «Сообщения ВКонтакте»
2. В сайдбаре проектов под фильтром команд появятся две новые секции: «Callback API» и «Диалоги»
3. Нажмите «Подключено», чтобы увидеть только проекты с настроенным callback
4. Нажмите «Не подключено», чтобы найти проекты без callback
5. Нажмите «Есть новые», чтобы увидеть только проекты с непрочитанными диалогами

**Что теперь можно:**  
Быстро находить проекты, которым нужно настроить callback. Быстро переключаться между проектами с непрочитанными сообщениями, не просматривая весь список.

### Фильтры отображения в чате

**Что добавили:**  
В шапке чата (рядом с поиском) появилась кнопка-воронка. По нажатию раскрывается панель с фильтрами отображения:
- **Сообщения:** показывать все / только наши / только пользователя
- **Скрыть вложения:** скрывает фото, документы, ссылки и другие вложения (показывает компактную подпись «N влож.»)
- **Скрыть кнопки:** скрывает кнопки бота (показывает подпись «кнопки»)

**Как использовать:**  
1. Откройте диалог с пользователем
2. Нажмите на иконку воронки в шапке чата (между поиском и обновлением)
3. Включите нужные фильтры: выберите тип сообщений, переключите «скрыть вложения» или «скрыть кнопки»
4. Когда фильтры активны, на иконке воронки появляется точка-индикатор
5. Чтобы сбросить всё — нажмите «Сбросить» в панели фильтров

**Что теперь можно:**  
Уменьшать визуальный шум в длинных диалогах с большим количеством вложений и кнопок бота. Быстро просматривать только текстовые сообщения для анализа переписки.

### Вкладка «Вложения» в профиле переписки

**Что добавили:**  
В правой панели с информацией о пользователе (рядом с вкладками «Профиль» и «Посты») появилась новая вкладка «Вложения». Она показывает все медиафайлы и ссылки из переписки, сгруппированные по категориям.

**Как использовать:**  
1. Откройте диалог с пользователем в модуле «Сообщения»
2. В правой панели нажмите вкладку «Вложения»
3. Выберите нужный подраздел: Фото / Видео / Ссылки / Файлы
4. Фото можно просмотреть (по клику) и скачать (кнопка при наведении)
5. Ссылки можно открыть или скопировать (кнопка копирования)
6. Файлы можно скачать по клику
7. Если загружена не вся история — нажмите «Загрузить все сообщения» для полного списка вложений

**Что теперь можно:**  
Быстро найти нужное фото, файл или ссылку из переписки без прокрутки всех сообщений. Скачать фотографию одним кликом. Скопировать ссылку из чата в буфер обмена.

### Автоматическое обновление списка рассылки при входящем сообщении

**Что добавили:**

**Как это работает:**  
1. Пользователь пишет сообщение в сообщество ВКонтакте
2. Система автоматически: собирает данные профиля пользователя из VK, добавляет его в список рассылки (или обновляет, если он уже есть)

4. В разделе «Списки → Рассылка» — таблица, статистика и мета-информация обновляются в реальном времени

**Что теперь можно:**  
Всегда иметь актуальный список подписчиков рассылки без ручных действий. Новые пользователи попадают в список мгновенно — не нужно ждать фоновой синхронизации или запускать обновление вручную.
