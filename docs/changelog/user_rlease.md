# Пользовательский релиз

Этот файл содержит пользовательские заметки о релизах: простым языком, без технических деталей.

---

## 🔴 Открытые задачи

### Сбой массовой синхронизации подписчиков

**Проблема:**  
При нажатии кнопки «Обновить подписчиков» в Админ-панели процесс синхронизации обрывается примерно на середине. Не все проекты успевают обновить списки подписчиков.

**Статус:** 🔴 В работе — проводится диагностика, ведётся поиск причины

---

## ✅ Решённые задачи

### Кнопка «Опубликовать сейчас» показывала ошибку, хотя пост публиковался

**Проблема:**  
При нажатии «Опубликовать сейчас» появлялась ошибка, а окно подтверждения зависало с индикатором загрузки. При этом пост на самом деле успешно появлялся на стене сообщества.

**Решение:**  
Исправлено — теперь кнопка корректно завершает работу, окно закрывается, и пост публикуется без ложных ошибок.

---

### Окно подтверждения публикации зависало после нажатия «Да»

**Проблема:**  
После нажатия «Да» в окне подтверждения публикации — пост публиковался, но само окно не закрывалось и продолжало показывать индикатор загрузки.

**Решение:**  
Исправлено — окно теперь корректно закрывается сразу после успешной публикации.

---

### Системный пост оставался в расписании после публикации

**Проблема:**  
При нажатии «Опубликовать сейчас» на системном посте — пост публиковался на стене, но его запись продолжала висеть в расписании вместо удаления.

**Решение:**  
Исправлено — после ручной публикации системный пост сразу удаляется из расписания. Циклические посты при этом продолжают работать как положено.

---

### Истории пропадали после массового удаления постов

**Проблема:**  
При выборе и удалении нескольких публикаций сразу — раздел расписания загружался без историй (stories). Они появлялись только после обновления страницы.

**Решение:**  
Исправлено — массовое удаление теперь корректно обновляет расписание, и все истории остаются на месте.

---

### Уведомления (тосты) занимали весь экран

**Проблема:**  
При массовых операциях (удаление постов, обновление публикаций) всплывающие уведомления растягивались на весь экран и перекрывали интерфейс. Не было возможности быстро закрыть все уведомления.

**Решение:**  
Теперь показывается не больше 5 уведомлений одновременно. Появилась кнопка «Скрыть все», чтобы закрыть все уведомления одним кликом. Если уведомлений больше 5, показывается счётчик скрытых.

---

### Фоновые задачи зависали на сервере

**Проблема:**  
Синхронизация подписчиков, массовое редактирование и публикация постов иногда зависали или завершались с ошибкой при работе на боевом сервере.

**Решение:**  
Исправлена внутренняя работа с базой данных — теперь фоновые задачи стабильно работают даже при параллельной обработке большого количества данных.

---

### Ошибки при мультипроектной публикации

**Проблема:**  
При сохранении поста сразу в несколько проектов часть постов не создавалась — возникала ошибка на сервере.

**Решение:**  
Исправлено — посты теперь создаются последовательно, что исключает конфликты. Все выбранные проекты получают свою копию поста.

### Ошибка при обновлении списка историй

**Проблема:**  
При обновлении списка историй (кнопка «Обновить список» в разделе «Посты в истории») иногда возникала внутренняя ошибка сервера.

**Решение:**  
Исправлена работа с данными историй — обновление теперь проходит стабильно, без ошибок.

---

### Медленное удаление отложенных постов

**Проблема:**  
Удаление отложенных постов иногда занимало заметно больше времени, чем ожидалось — система перебирала лишние аккаунты.

**Решение:**  
Оптимизировано — теперь при удалении отложенного поста система сразу использует аккаунт-администратор группы, что ускоряет процесс.

### Истории показывались как архивные при входе в раздел

**Проблема:**  
При входе в раздел «Статистика историй» все истории отображались как архивные. После нажатия «Обновить список» активные истории корректно показывались.

**Решение:**  
Исправлено — теперь статус историй (активная/архивная) сохраняется и корректно отображается сразу при загрузке страницы. Кнопка «Обновить список» по-прежнему получает свежие данные из ВКонтакте.

### Бесконечная загрузка в разделе «Промокоды»

**Проблема:**

**Решение:**  
Исправлено — раздел промокодов теперь загружается корректно с первого раза.

---

### Статус промокодов не обновлялся в боковом меню

**Проблема:**  
После добавления или удаления промокодов в разделе «Конкурс отзывов» статус в боковом меню не менялся. Приходилось перезагружать страницу.

**Решение:**  
Исправлено — теперь при любых изменениях промокодов статус в боковом меню обновляется автоматически.

## 🆕 Новые возможности

### Массовое редактирование постов

**Что добавили:**  
Возможность найти одинаковые посты по всем проектам и отредактировать их все разом — изменить текст, удалить изображения или сменить дату публикации.

**Как использовать:**  
1. Откройте любой пост и нажмите кнопку массового редактирования
2. Настройте критерии поиска: по тексту и/или по дате, выберите проекты
3. Нажмите «Найти» — система покажет все совпадения
4. Отметьте нужные посты, внесите изменения (текст, изображения, дата)
5. Нажмите «Применить» — изменения будут применены ко всем выбранным постам

**Что теперь можно:**  
Вместо того чтобы вручную редактировать один и тот же пост в 10 проектах, достаточно сделать это один раз. Система сама найдёт все копии и применит правки. Прогресс отображается в реальном времени, а по завершении показывается отчёт с результатами.

---

### Сдвиг времени при мультипроектной публикации

**Что добавили:**  
При создании поста сразу в несколько проектов теперь можно задать интервал между публикациями — чтобы посты выходили не одновременно, а с задержкой.

**Как использовать:**  
1. При создании поста выберите несколько проектов
2. Включите переключатель «Сдвиг по времени между проектами»
3. Задайте интервал: дни, часы, минуты (например, 10 минут)
4. Перетащите проекты в нужном порядке публикации
5. В таблице внизу увидите рассчитанное время для каждого проекта
6. Сохраните — посты будут запланированы с указанным сдвигом

**Что теперь можно:**  
Публиковать один пост в 10 сообществах с интервалом 10 минут между ними — вместо того, чтобы вручную менять время для каждого проекта. Порядок публикации можно настроить перетаскиванием.

---

### Улучшенная подсказка при отсутствии авторизованных аккаунтов

**Что добавили:**  
Вместо непонятной «Ошибки доступа» на вкладках «Расписание» и «Предложенные посты» теперь показывается подробная пошаговая инструкция, как подключить администратора сообщества.

**Как использовать:**  
1. Если вы видите баннер «Нет авторизованных аккаунтов» — раскройте блок «Подробнее — как это исправить»
2. Следуйте пошаговой инструкции: перейдите в Настройки → Системные страницы
3. Добавьте ссылку на профиль ВК-администратора и авторизуйте его через ключ

**Что теперь можно:**  
Самостоятельно решить проблему с доступом, не обращаясь за помощью — инструкция проведёт по всем шагам.

---

### Инструкция по получению токена сообщества

**Что добавили:**  
В настройках проекта (вкладка «Интеграции») появилась встроенная инструкция, как получить токен сообщества ВКонтакте.

**Как использовать:**  
1. Откройте настройки проекта → вкладка «Интеграции»
2. Найдите информационный блок с инструкцией
3. Перейдите по ссылке в настройки сообщества ВК (Управление → Работа с API → Ключи доступа)
4. Создайте токен с нужными правами и вставьте его в поле

**Что теперь можно:**  
Получить токен сообщества без посторонней помощи — инструкция покажет, где именно его найти в ВКонтакте и какие права нужно выбрать.

## ⚠️ Известные ограничения

Этот раздел содержит информацию о текущих ограничениях, которые могут влиять на работу.

### Массовое редактирование: скорость и капча

- **При редактировании 50+ постов** процесс может занять несколько минут — ВКонтакте ограничивает скорость запросов (не более 3 в секунду). Дождитесь завершения, не закрывайте окно.
- **Некоторые посты могут не отредактироваться** — ВКонтакте может запросить капчу при массовых операциях. Попробуйте повторно или обновите токен в настройках.
- **Поиск не находит пост** — поиск работает по точному совпадению текста. Попробуйте искать только по дате.

### Сдвиг времени: скорость сохранения

- **Сохранение поста в 10+ проектов** может занять 10-20 секунд — посты создаются последовательно для надёжности. Не закрывайте окно до завершения.
- **Перетаскивание проектов** может не работать на мобильных устройствах — используйте десктопную версию.

### Фоновые задачи: длительные операции

- **Операции длительностью >15 минут** могут завершиться с ошибкой соединения — облачная база данных закрывает неактивные соединения. Запустите повторно — система выполнит до 3 попыток переподключения.

### Публикация: дублирование при одновременном срабатывании

- **Если нажать «Опубликовать сейчас» точно в момент** запланированной автоматической публикации — пост может опубликоваться дважды. Удалите дубликат вручную со стены сообщества.
