### Счётчик важных диалогов на звёздочке

**Что добавили:**  
Теперь на иконке-звёздочке в панели диалогов отображается счётчик — сколько диалогов помечено как важные. Янтарный кружок с числом, аналогичный синему счётчику непрочитанных.

**Как использовать:**  
1. Зайдите в раздел «Сообщения» и выберите проект
2. На панели фильтров увидите звёздочку с числом — это количество важных диалогов
3. Нажмите на звёздочку — отобразятся только важные диалоги

**Что теперь можно:**  
Быстро видеть, сколько диалогов помечено важными, не переключая фильтр. Нажатие на звёздочку теперь реально фильтрует список, показывая только важные диалоги.

### «В админы» — включение основной страницы (ENV-токен)

**Что добавили:**  
В модалке «В админы» появился чекбокс «+ ENV токен», который позволяет включить вашу основную страницу (из настроек .env) в процесс вступления в группы и назначения администратором — наравне с системными страницами.

**Как использовать:**  
1. Откройте «Управление базой проектов» → кнопка «В админы».
2. Выберите проекты (группы VK) в левой колонке.
3. Выберите системные страницы в правой колонке (при необходимости).
4. В футере модалки поставьте галочку **«+ ENV токен»** — основная страница будет добавлена к списку.
5. Нажмите «В админы».

**Что теперь можно:**  
Одним кликом добавить основную страницу из .env в процесс массового вступления и назначения админом, не добавляя её отдельно как системную страницу.

### Информация VK — VK ID, ссылка по ID, кнопки копирования

**Что добавили:**  
В настройках проекта (раздел «Информация VK») теперь отображается VK ID сообщества и ссылка на группу в формате с ID. Рядом с каждым полем и ссылкой появилась кнопка копирования — можно скопировать данные в один клик.

**Как использовать:**  
1. Откройте настройки проекта (иконка шестерёнки на карточке проекта).
2. Раскройте секцию «Информация VK» в нижней части.
3. Вы увидите 4 поля: Название группы VK, VK ID, Ссылка на проект (shortname), Ссылка по VK ID.
4. Нажмите кнопку копирования (📋) рядом с любым полем — данные скопируются в буфер обмена.
5. При успешном копировании кнопка ненадолго покажет галочку ✅.
6. У ссылок также сохранились кнопки перехода (↗) для открытия в новой вкладке.

**Что теперь можно:**  
Быстро скопировать VK ID, ссылку или название группы из настроек проекта — без ручного выделения текста. Ссылка по VK ID полезна, когда shortname группы не работает или сменился.

### Предпросмотр поста — просмотр всех фото с навигацией

**Что добавили:**  
Теперь в предпросмотре поста можно кликнуть на любое фото и оно откроется в полноэкранном режиме. Если к посту прикреплено больше 4 фото — все они доступны для просмотра через стрелки навигации.

**Как использовать:**  
1. Откройте пост (кликните на карточку в расписании).
2. В предпросмотре справа кликните на любое фото — оно откроется на весь экран.
3. Используйте стрелки ← → (кнопки или клавиши клавиатуры) для переключения между фото.
4. Внизу отображается счётчик «X / Y» — текущее фото из общего числа.
5. Нажмите Escape или кликните на тёмный фон для закрытия.

**Что теперь можно:**  
Просматривать ВСЕ прикреплённые фотографии к посту (не только первые 4), проверять качество и порядок перед публикацией.

### Системные списки — подробная информация о постах при раскрытии пользователя

**Что добавили:**

**Что теперь можно:**  
Сразу видеть, с какими именно постами взаимодействовал пользователь: какой текст, сколько реакций, какая картинка — без необходимости открывать каждый пост в VK отдельно.

### Датапикер — быстрый выбор месяца и года

**Что добавили:**  
В календаре (выбор даты) теперь можно быстро переключить месяц и год без пролистывания стрелками. Работает во всех местах приложения, где используется выбор даты.

**Как использовать:**  
1. Откройте любой календарь (выбор даты).
2. Кликните на заголовок месяца (например, «Март 2026») — откроется выбор месяца.
3. Кликните на год — откроется выбор года (десятилетие).
4. Стрелки ← → переключают: месяцы вперёд/назад (в режиме дней), годы (в режиме месяцев), десятилетия (в режиме годов).
5. Выберите нужный год → месяц → день.

**Что теперь можно:**  
Перейти с марта 2026 на март 2021 за 3 клика, а не за 60 пролистываний стрелками. Удобно при работе со старыми заказами, статистикой или архивными данными.

### Сопутствующие товары — аналитика «что покупают вместе»

**Что добавили:**

**Как использовать:**  
1. Откройте статистику DLVRY → таб «Товары».
2. Кликните на любую строку товара.
3. Под ней появится блок с карточками: процент совместных заказов, кол-во заказов, ср. кол-во штук.
4. Повторный клик — скрывает дашборд.

**Что теперь можно:**  
Понять, какие товары клиенты чаще всего берут вместе. Например, если ролл Филадельфия есть в 1000 заказах, в 50% из них также есть ролл Калифорния — это помогает составлять комбо-наборы и продвигать товары.

### Хронология действий менеджера в чате

**Проблема:**  
В диалогах раздела «Сообщества — Сообщения» не отображались действия менеджеров: кто зашёл в диалог, кто пометил важным, кто прикрепил метку, кто переслал сообщения в групповой чат. Информация терялась.

**Что сделано:**  
Теперь все действия менеджеров отображаются прямо в хронологии чата мелким текстом — между обычными сообщениями. Действия включают: вход и выход из диалога, пометка важным / снятие, пометка непрочитанным, прикрепление и снятие меток, пересылка сообщений в групповой чат. Информация появляется мгновенно в реальном времени.

⚠️ **Протестировать после развёртывания**: зайти в диалог → выполнить любое действие (пометить важным, прикрепить метку, переслать в группу) → убедиться что запись появилась в чате.

### Шаблон из чата оставался при переключении диалога

**Проблема:**  
После использования шаблона в чате, при переходе в другой проект или к другому клиенту, текст шаблона оставался в поле ввода. Также могли сохраняться прикреплённые файлы и открытые панели.

### Исправлено ложное «Сохранено» при массовом обновлении проектов

**Проблема:**  
При сохранении изменений в управлении проектами, если один из проектов был удалён другим пользователем, система показывала «Сохранено!», хотя часть данных не обновилась.

**Решение:**  
Теперь если какой-то проект не удалось найти в базе — система покажет ошибку с указанием, какие именно проекты не обновились.

---

### Улучшена валидация данных контекста проекта

**Проблема:**  
При сохранении контекста проекта сервер мог принять некорректные данные без проверки, что приводило к непонятным ошибкам.

**Решение:**  
Добавлена строгая проверка формата данных на сервере. Теперь если что-то не так — сервер вернёт понятное сообщение об ошибке вместо технического сбоя.

### Групповые чаты (беседы) ВКонтакте

**Что добавили:**  
В разделе «Сообщения» появилась возможность просматривать и писать в групповые чаты (беседы) сообщества. Раньше были доступны только личные диалоги.

**Как использовать:**  
1. Откройте «Сообщества → Сообщения».
2. Нажмите кнопку с иконкой группы людей (третья в панели фильтров) — «Чаты».
3. Появится список групповых чатов. Если пусто — нажмите «Синхронизировать».
4. Выберите чат — откроется история сообщений.

**Что теперь можно:**  
Читать и отвечать в групповых чатах (беседах) сообщества прямо из планировщика, не переключаясь в ВК.

### Имена отправителей в групповых чатах

**Что добавили:**  
В групповых чатах теперь видно, кто отправил каждое сообщение — над текстом отображается цветное имя отправителя.

**Как использовать:**  
Просто откройте любой групповой чат — имена отображаются автоматически для всех входящих сообщений.

**Что теперь можно:**  
Понимать, кто именно написал сообщение в групповом чате, не переходя в ВК.

### Убран ложный бейдж непрочитанных на кнопке «Чаты»

**Проблема:**  
На кнопке «Чаты» отображался бейдж с цифрой, который выглядел как счётчик непрочитанных сообщений, хотя на самом деле показывал количество чатов.

### Устранены массовые ошибки при загрузке приложения

**Проблема:**  
При входе в приложение (особенно в раздел «Сообщения») при большом количестве проектов (150+) происходили каскадные ошибки загрузки — данные не подгружались с первого раза.

**Решение:**  
Оптимизирована стратегия загрузки — снижено количество одновременных запросов к серверу, убраны лишние повторные попытки, улучшена обработка ошибок на сервере. Загрузка стала стабильнее и быстрее.
