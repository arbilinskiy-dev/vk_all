## ✅ Решённые задачи

### Данные проектов не загружались (ошибка 502)

**Проблема:**  
После входа в приложение данные проектов (посты, отложенные записи, истории и т.д.) не загружались — вместо этого отображались пустые списки или ошибки. Особенно проявлялось при большом количестве проектов (~145+).

**Решение:**  
Полностью переработана архитектура загрузки данных. Вместо одного тяжёлого запроса, данные теперь загружаются параллельно небольшими порциями. Это устранило ошибки и значительно ускорило начальную загрузку приложения:
- Интерфейс становится доступен сразу после загрузки списка проектов (1–2 секунды).
- Остальные данные (посты, расписание, истории) подгружаются в фоне порциями.
- Все 7 типов данных теперь загружаются стабильно.

### Токены сообществ переставали работать

**Проблема:**  
После обновления серверной части приложения все привязанные аккаунты и токены сообществ отображались как «ошибка». При проверке токена система сообщала: «Токен недействителен». Работал только один аккаунт, а остальные (~136) — нет.

**Решение:**  
Выявлена и устранена проблема рассинхронизации при обновлении серверной инфраструктуры. Все 136 затронутых токенов были восстановлены автоматически, пользователям не нужно заново привязывать аккаунты. Дополнительно созданы диагностические утилиты для быстрого обнаружения подобных проблем в будущем.

### Ошибка создания товара с фото-заглушкой

**Проблема:**  
При создании нового товара с включённой галочкой «Заглушка (дефолтное фото)» появлялась ошибка: «Изображение слишком маленькое. VK требует минимум 400x400 пикселей». Товар не создавался.

### Индивидуальные даты и время для каждого проекта в мультипубликации

**Что добавили:**  
Теперь при мультипроектной публикации можно задать свою дату и время для каждого проекта по отдельности, а не только одну общую дату для всех.

**Как использовать:**  
1. Создайте новый пост и включите переключатель «Мультипроектная публикация».
2. Выберите нужные проекты (2 и более).
3. Появится секция **«Расписание публикации»** — список проектов с полями даты и времени.
4. Измените дату или время у любого проекта — поле подсветится фиолетовым.
5. Чтобы вернуть автоматическую дату, нажмите кнопку ↻ рядом с проектом.
6. При необходимости включите «Автоматический сдвиг времени» — он автоматически разведёт даты с заданным интервалом.
7. Перетаскивайте проекты для изменения порядка публикации.

**Что теперь можно:**  
- Запланировать один и тот же пост в 10 проектов на разные даты и время — каждый проект настраивается индивидуально.
- Комбинировать: автоматический сдвиг задаёт базовое расписание, а затем можно подправить конкретные проекты вручную.
- В сводке перед сохранением видно итоговое расписание для каждого проекта.

### Мульти-команды: несколько команд для одного проекта

**Что добавили:**  
Теперь одному проекту можно присвоить сразу несколько команд. Раньше каждый проект мог принадлежать только одной команде — теперь можно выбрать сколько угодно.

**Как использовать:**  
1. Откройте настройки проекта (шестерёнка на карточке проекта).
2. В разделе **«Команды»** нажимайте на нужные команды — они выделятся. Нажмите повторно, чтобы убрать.
3. Над кнопками команд отображается текущий выбор: «Выбрано: Команда А, Сеть Н».
4. Кнопка **«Сбросить все»** убирает все команды.
5. Добавить новую команду можно как раньше — через поле внизу.

**Что теперь можно:**  
- Присвоить проекту несколько команд одновременно (например, «Команда А» и «Сеть Н»).
- В сайдбаре при фильтрации по команде отображаются все проекты, которые входят в эту команду (даже если у проекта несколько команд).
- В разделе «Управление базой проектов» массовое редактирование команд тоже поддерживает множественный выбор с чекбоксами.

### Отмена и повтор действий в текстовом поле (Undo/Redo)

**Что добавили:**  
Теперь в текстовом поле поста работает полноценная отмена и повтор действий — как клавишами, так и кнопками.

**Как использовать:**  
1. Откройте редактирование поста — над текстовым полем появились кнопки **«Отменить»** и **«Повторить»**.
2. Введите текст, вставьте AI-генерацию или эмодзи.
3. Нажмите **Ctrl+Z** или кнопку **«Отменить»** — текст вернётся к предыдущему состоянию.
4. Нажмите **Ctrl+Shift+Z** или кнопку **«Повторить»** — текст восстановится.
5. Наведите на иконку ⓘ рядом с кнопками — появится подсказка с горячими клавишами.

**Что теперь можно:**  
Отменять любые изменения текста — включая вставку AI-текста, эмодзи и переменных. Раньше после вставки сгенерированного текста Ctrl+Z не работал.

### Мультипроектная публикация игнорировала даты при создании нескольких постов

**Проблема:**  
При одновременном включении «Создать несколько постов» и «Мультипроектная публикация» все посты для каждого проекта создавались на одну и ту же дату, хотя в настройках были указаны разные даты (например, 14.02, 15.02, 16.02).

**Решение:**  
Исправлена логика расчёта дат. Теперь каждый проект получает свою дату с учётом сдвига времени. Например, при 3 проектах и 3 датах создаётся 9 постов, каждый с уникальной датой и временем. В сводке «Будет запланировано» теперь отображается реальное количество постов и точные даты для каждого проекта.

### Автоматизация историй создавала дубликаты

**Проблема:**  
Автоматическая публикация историй из постов создавала 4 одинаковые истории вместо одной. Например, из одного поста с фотографией появлялось сразу 4 копии истории в одно и то же время.

**Решение:**  
Исправлена ошибка, из-за которой при одновременной работе нескольких серверных процессов каждый из них создавал свою копию истории. Теперь добавлена тройная защита: блокировка на уровне процесса, уникальное ограничение в базе данных и автоматическая обработка конфликтов. Гарантируется ровно 1 история на 1 пост.

### Новое окно создания и редактирования поста

**Что добавили:**

**Что теперь можно:**  
Работать со всеми инструментами одновременно — текст, AI-генерация, настройки и предпросмотр видны на одном экране. Каждая колонка прокручивается независимо.

**Что теперь можно:**

### Ускорен сбор статистики историй

**Что добавили:**  
Оптимизирована работа сбора статистики и зрителей историй. Система теперь автоматически определяет архивные истории, чья статистика уже не меняется, и пропускает их при следующих обновлениях. «Мёртвые» истории (удалённые из VK или недоступные) больше не опрашиваются бесконечно.

**Как это работает:**  
1. При обновлении статистики система сравнивает новые данные с уже сохранёнными.
2. Если данные архивной истории не изменились — история помечается как «финализированная» и пропускается в дальнейшем.
3. Удалённые или недоступные истории автоматически помечаются при первой же ошибке.
4. Если архивная история снова станет активной — пометка автоматически снимается.

**Что теперь можно:**  
Обновление статистики проектов с большим количеством историй (100+) стало в разы быстрее. Вместо опроса всех историй каждый раз, система обрабатывает только те, чьи данные ещё могут измениться. Например, проект со 176 историями теперь обрабатывается за ~4-6 запросов вместо ~352.
