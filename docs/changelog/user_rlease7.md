## 🆕 Новые возможности

### Статус «можно ли писать» обновляется в реальном времени

**Что добавили:**  
Когда пользователь VK разрешает или запрещает сообщения сообщества, статус «Можно писать» / «Нельзя писать» теперь обновляется мгновенно прямо в открытом диалоге — без перезагрузки страницы. Также этот статус теперь отображается в шапке чата рядом с именем пользователя.

**Как использовать:**  
1. Откройте модуль «Сообщения» и выберите диалог с пользователем
2. В шапке чата (рядом с именем) виден badge — зелёный «Можно писать» или красный «Нельзя писать»
3. Такой же badge есть в правой панели информации о пользователе
4. Если пользователь запретит сообщения — оба badge мгновенно станут красными
5. Если снова разрешит — оба badge станут зелёными

**Что теперь можно:**  
- Видеть актуальный статус «можно ли писать» прямо в шапке чата, не открывая правую панель
- Получать обновление статуса мгновенно (по VK Callback), не перезагружая страницу
- Понимать, почему сообщения не доставляются конкретному пользователю

### Мониторинг активных сессий — «Кто в сети»

**Что добавили:**  
На странице «Управление пользователями» появилась новая вкладка «Активные сессии». Она показывает, кто из сотрудников прямо сейчас работает в системе: имя, статус активности (цветной индикатор), IP-адрес, браузер, время входа и длительность сессии. Администратор может принудительно завершить любую сессию — например, если сотрудник забыл выйти на чужом компьютере.

**Как использовать:**  
1. Перейдите на страницу «Управление пользователями» через боковое меню
2. Нажмите на вкладку «Активные сессии» (доступна только администраторам)
3. Вы увидите таблицу всех пользователей, которые сейчас авторизованы
4. Зелёный кружок — пользователь активен, жёлтый — отошёл, серый — неактивен
5. Чтобы завершить чью-то сессию, нажмите иконку выхода справа и подтвердите действие
6. Данные обновляются автоматически каждые 30 секунд, или нажмите «Обновить» вручную

**Что теперь можно:**  
- Видеть, кто из сотрудников сейчас работает в системе в реальном времени
- Контролировать количество одновременных сессий
- Принудительно завершать сессии (если сотрудник забыл выйти)
- Отслеживать с каких IP-адресов и браузеров заходят пользователи

### Бесконечная прокрутка для Callback логов

**Что добавили:**  
Таблица логов Callback API теперь загружает данные порциями по 50 записей — как на странице «VK Логи». При прокрутке вниз автоматически подгружаются следующие записи. Это ускоряет открытие страницы и экономит ресурсы.

**Как использовать:**  
1. Откройте «Настройки» → вкладка «Колбэк логи»
2. Первые 50 записей загрузятся сразу
3. Прокрутите таблицу вниз — при достижении конца автоматически подгрузятся следующие 50
4. Внизу таблицы появится индикатор загрузки, а когда все записи загружены — соответствующее сообщение

**Что теперь можно:**  
- Быстрее открывать страницу логов (не нужно ждать загрузки всех записей)
- Видеть сколько именно записей загружено из общего количества в БД
- Плавно просматривать большие объёмы логов

### Поддержка домена vk.ru

**Что добавили:**  
Теперь система принимает ссылки как с домена `vk.com`, так и с `vk.ru`. Раньше, если вставить ссылку вида `https://vk.ru/club236229266`, система не могла определить сообщество. Теперь оба формата работают одинаково — в любом месте, где нужно вставить ссылку на сообщество, пользователя или сториз.

3. Система автоматически распознает ссылку и извлечёт нужный идентификатор

**Что теперь можно:**  
- Вставлять ссылки с `vk.ru` при добавлении проектов
- Использовать ссылки `vk.ru` в настройках конкурсов и автоматизаций
- Вставлять ссылки на сториз с `vk.ru` для автоматизации сториз
- Искать пользователей в списках по ссылке `vk.ru`
- Все placeholder'ы и подсказки обновлены и показывают оба домена

### Интерактивный график мониторинга сообщений с детальными метриками

### Автоматическое увеличение маленьких фото товаров

**Что добавили:**  
При загрузке товаров фото размером менее 400×400 пикселей теперь автоматически увеличиваются до минимально допустимого размера VK. Раньше такие фото вызывали ошибку, и их приходилось вручную увеличивать через внешние сервисы.

4. Перейдите в VK и проверьте качество увеличенных фото — при необходимости замените на фото большего размера

**Что теперь можно:**  
- Загружать товары с любыми фото без ошибок — маленькие фото увеличиваются автоматически
- Получать предупреждение о том, что качество увеличенного фото может быть ниже
- Не тратить время на ручное изменение размеров через внешние сервисы

### Истории по постам публикуются мгновенно

**Что изменилось:**  
Раньше автоматизация «Пост → История» проверяла новые посты каждые 10 минут, из-за чего история могла появиться с задержкой до 10 минут после публикации поста. Теперь история публикуется мгновенно — в течение 3 секунд после появления нового поста.

**Как это работает:**  
1. Вы публикуете пост в сообществе VK (или через планер)
2. Система получает уведомление от VK и сразу создаёт историю — без ожидания
3. Раз в час проходит проверка «на всякий случай» — если какой-то пост был пропущен, история будет создана при этой проверке
4. Раз в час собирается статистика по опубликованным историям — просмотры, зрители

**Что теперь можно:**  
- Публиковать пост и видеть историю в VK практически мгновенно (3 секунды вместо 10 минут)
- Не беспокоиться о пропущенных постах — система перепроверяет раз в час
- Статистика по историям и данные о зрителях собираются автоматически

**Как использовать:**  
Никаких дополнительных действий не требуется. Если автоматизация «Пост → История» была активна — она продолжит работать, только быстрее. Настройки (ключевые слова, шаблон истории) сохраняются без изменений.

### Настройка интеграции прямо на странице автоматизации историй

**Что добавили:**  
На странице «Автоматизация историй → Настройки» появился блок проверки готовности интеграции. Система проверяет 3 требования (токен, Callback-сервер, событие wall_post_new) и позволяет настроить каждое прямо здесь — без перехода в настройки проекта. Включение автоматизации заблокировано, пока все 3 шага не выполнены.

**Как использовать:**  
1. Откройте «Автоматизация историй» → вкладка «Настройки»
2. Вверху страницы отображается чеклист из 3 пунктов с зелёными галочками или красными крестиками
3. Если нет токена — введите его прямо в форму и нажмите «Сохранить»
4. Если нет Callback-сервера — нажмите «Настроить автоматически» (тот же интерфейс, что в настройках проекта)
5. Если событие wall_post_new отключено — нажмите «Включить wall_post_new»
6. Когда все 3 пункта зелёные — тогл включения автоматизации разблокируется

**Что теперь можно:**  
- Настроить всю интеграцию для автоматизации историй, не покидая страницу автоматизации
- Увидеть текущее состояние подключения (токен, Callback-сервер, события) одним взглядом
- Быстро диагностировать, почему автоматизация не работает — видно какой именно шаг пропущен
