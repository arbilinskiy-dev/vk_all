# Архитектурные Reference-файлы

Опорная база паттернов, анти-паттернов и рецептов, выработанных на реальных инцидентах проекта. Каждый файл — это **воспроизводимый рецепт** для лечения конкретного класса проблем.

## Отличие от дизайн-системы (uiux-refactor/references/)

| | Архитектурные references | Дизайн-система references |
|---|---|---|
| **Фокус** | Производительность, рендеринг, data-flow, стейт-менеджмент | Визуал, верстка, анимации, цвета, отступы |
| **Уровень** | React hooks, memo, context, state | JSX, CSS, компоненты UI-кита |
| **Пример** | useCallback+ref паттерн, hasChanges merge | Кнопки, дропдауны, табы, SVG-чарты |
| **Триггер** | Тормоза, лишние рендеры, зависания | Визуальное несоответствие макету |

## Когда создавать новый reference

1. Обнаружен повторяющийся класс проблем (>1 инцидента одного типа)
2. Решение нетривиальное и содержит >3 шагов
3. Есть антипаттерны, которые легко повторить
4. Есть конкретный «до/после» с измеримым результатом
