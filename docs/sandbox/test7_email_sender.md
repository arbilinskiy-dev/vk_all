# Тест 7: Массовая отправка email с вложениями (Яндекс SMTP)

## Статус: ✅ Работает (03.03.2026)

---

## Цель

Отправка бизнес-документов контрагентам через Яндекс SMTP:
- Каждому контрагенту — своё письмо с индивидуальной темой
- Письмо уходит на несколько email-адресов (разные департаменты)
- Поддержка вложений (PDF, DOCX и т.д.)
- Копия сохраняется в «Отправленные» через IMAP

## Сценарий использования

1. У пользователя есть **список контрагентов** (до 100 компаний)
2. У каждого контрагента **до 5 email** (бухгалтерия, директор, закупки и т.д.)
3. Каждому нужно отправить **одно письмо** с вложением (акт, счёт и т.д.)
4. Система берёт CSV, формирует письма и отправляет с паузами

## Полная инструкция: подготовка Яндекс-аккаунта

### Шаг 1. Включить доступ по IMAP

1. Зайти в Яндекс Почту: https://mail.yandex.ru
2. Нажать **⚙ Настройки** (шестерёнка) → **Все настройки**
3. Выбрать раздел **«Почтовые программы»** (или напрямую: https://mail.yandex.ru/?dpda=yes#setup/client)
4. Поставить галочку: **«С сервера imap.yandex.ru по протоколу IMAP»**
5. Убедиться, что включена опция: **«Пароли приложений и OAuth-токены»**
6. Нажать **Сохранить**

### Шаг 2. Создать пароль приложения

1. Перейти: https://id.yandex.ru/security/app-passwords
2. В разделе **«Создать пароль приложения»** нажать **[+]** рядом с **«Почта»** (IMAP, POP3, SMTP)
3. Ввести название, например: `email-sender`
4. Нажать **Далее**
5. **Скопировать пароль** (показывается только один раз!)
6. Этот пароль использовать в поле «Пароль приложения» в тесте

> ⚠️ **Важно:** Основной пароль от Яндекс-аккаунта **НЕ подойдёт**. Нужен именно пароль приложения.

### Шаг 3. Проверить настройки аккаунта

- Двухфакторная аутентификация: если включена — пароль приложения **обязателен** (основной точно не сработает)
- Если аккаунт новый или давно не использовался — войти через браузер хотя бы раз, чтобы подтвердить активность

## Как пользоваться тестом

### Запуск

1. Запустить бэкенд: `cd backend_python && python main.py`
2. Запустить фронтенд: `npm run dev` (или `npx vite`)
3. Открыть: http://localhost:5173 → **Песочница** → вкладка **«Тест 7: Email-рассылка»**

### Вкладка «Подключение»

Первым делом проверить соединение:
1. Ввести **логин** — полный email, например `bilinskiyar@yandex.ru`
2. Ввести **пароль приложения** (из Шага 2)
3. Оставить галочку **SSL (порт 465)** — рекомендуется
4. Нажать **«Проверить подключение к SMTP»**
5. Ожидаемый результат: ✅ «Подключение успешно!»

**Типичные ошибки:**
| Ошибка | Причина | Решение |
|---|---|---|
| `Invalid user or password` | Неверный логин или пароль | Проверить: используете пароль приложения (не основной), логин с @yandex.ru |
| `Connection refused` | Порт заблокирован | Попробовать снять галку SSL (порт 587) |
| `Connection timed out` | Файрвол / VPN | Проверить сетевое подключение |

### Вкладка «Одиночная отправка»

Тестовая отправка одного письма:
1. Ввести **«Кому»** — email получателя (можно несколько через запятую)
2. Ввести **Тему** письма
3. Отредактировать **тело письма** (HTML)
4. **(Опционально)** Прикрепить файл — PDF, DOCX, XLSX и т.д. (до 25 МБ)
5. Нажать **«Отправить письмо»**

Поддерживаемые форматы вложений: PDF, DOCX, XLSX, ZIP, RAR, PNG, JPG, CSV, TXT — любые файлы. MIME-тип определяется автоматически по расширению.

### Вкладка «Массовая отправка»

Отправка разных писем разным контрагентам:

**1. Подготовить CSV**

Формат (разделитель — точка с запятой):
```
contractor;emails;subject
ООО Ромашка;ivan@romashka.ru,buh@romashka.ru,dir@romashka.ru;Акт сверки за февраль 2026
ООО Василёк;dir@vasilek.ru,office@vasilek.ru;Счёт-фактура №123
ИП Иванов;ivanov@gmail.com;Закрывающие документы за Q1
```

Правила:
- Первая строка — заголовок: `contractor;emails;subject`
- Разделитель колонок: `;` (точка с запятой)
- Email-ы внутри ячейки: через `,` (запятую)
- Каждая строка — один контрагент

**2. Вставить CSV** в текстовое поле и нажать **«Предпросмотр CSV»** — появится таблица для проверки

**3. Настроить шаблон письма** (HTML)

Доступные плейсхолдеры:
- `{contractor}` — подставится имя контрагента из CSV
- `{subject}` — подставится тема из CSV

Пример шаблона:
```html
<p>Добрый день!</p>
<p>Направляем вам: <b>{subject}</b></p>
<p>Документ во вложении.</p>
<br>
<p>С уважением,<br>Ваша компания</p>
```

**4. Прикрепить вложение** (один файл для всех контрагентов)

**5. Настроить задержку** между письмами (рекомендация: 5–10 сек)

**6. Нажать «Запустить массовую отправку»** и дождаться результата

### Тест 4: Диагностика IMAP — папка «Отправленные» (03.03.2026)
- **Статус:** ✅ Пройден
- **Проблема:** Письма уходили получателю, но НЕ появлялись в «Отправленных» у отправителя
- **Причина:** Код перебирал имена папок наугад (`"Sent"`, `"INBOX.Sent"`, `"&BB4EQ..."`). Первый неверный APPEND ломал IMAP-сессию → последующие тоже не срабатывали
- **Диагностика IMAP LIST:**
  ```
  (\Sent) "|" "&BB4EQgQ,BEAEMAQyBDsENQQ9BD0ESwQ1-"   ← Отправленные
  (\Drafts) "|" "&BCcENQRABD0EPgQyBDgEOgQ4-"           ← Черновики
  (\Junk) "|" "&BCEEPwQwBDw-"                           ← Спам
  (\Trash) "|" "&BCMENAQwBDsENQQ9BD0ESwQ1-"             ← Удалённые
  INBOX                                                  ← Входящие
  ```
- **Ключевое:** Яндекс НЕ использует `"Sent"` — папка называется `&BB4EQgQ,BEAEMAQyBDsENQQ9BD0ESwQ1-` (IMAP UTF-7 кодировка «Отправленные»)
- **SELECT подтверждение:** `OK, 1702 сообщений`

### Тест 5: IMAP APPEND в «Отправленные» — изолированный (03.03.2026)
- **Статус:** ✅ Пройден
- **Метод:** Прямой `imap.append()` в найденную папку `&BB4EQgQ,BEAEMAQyBDsENQQ9BD0ESwQ1-`
- **Результат:** `('OK', [b'[APPENDUID 1462099325 1730] APPEND Completed.'])`
- **Проверка:** Письмо появилось в «Отправленных» в веб-интерфейсе Яндекс Почты ✅

### Найденные проблемы и решения

| # | Проблема | Причина | Решение |
|---|---|---|---|
| 1 | Ошибка авторизации | Опечатка в домене (`ynadex` вместо `yandex`) | Исправить логин |
| 2 | Нет в «Отправленных» | SMTP не сохраняет копию автоматически | Добавлен IMAP APPEND после отправки |
| 3 | IMAP APPEND не срабатывал | Перебор несуществующих папок (`Sent`, `INBOX.Sent`) ломал сессию | Поиск папки с атрибутом `\Sent` через `LIST`, один точный APPEND |
| 4 | Вложение как `octet-stream` | Не определялся MIME-тип | Автоопределение по расширению (`mimetypes.guess_type`) |
| 5 | Кириллица в имени файла | Некорректное кодирование | RFC 2231 encoding: `("utf-8", "", filename)` |
| 6 | Нет служебных заголовков | Отсутствовали Date / Message-ID | Добавлены `formatdate()` и `make_msgid()` |
| 7 | Нет обратной связи о сохранении | API не сообщал, сохранено ли в «Отправленные» | Добавлено поле `saved_to_sent` в ответ |
