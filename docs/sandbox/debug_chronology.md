## Итерация 11: Тест 4 — Первая проверка моделей Google AI

- **Цель:** Проверить доступность ~27 моделей Google AI по API-ключу.
- **Цепочка:** `POST /v1beta/models/{id}:generateContent` / `:embedContent` / `:predict` / `GET /models/{id}`
- **Результат:** ❌ Только 8/27 доступны. Большинство вернули 404 — неправильные model_id (взяты из документации, а не из реального API).

- **Решение:** Вызван `GET /v1beta/models?pageSize=100` для получения актуального списка. Обновлено до 36 моделей с корректными ID.

## Итерация 12: Исправление кодировки фронтенда

- **Проблема:** После обновления GeminiModelsTest.tsx через PowerShell `Set-Content -Encoding UTF8` файл стал нечитаемым — BOM-маркер + двойное кодирование кириллицы.
- **Причина:** PowerShell 5.1 `Set-Content -Encoding UTF8` добавляет BOM (0xEF 0xBB 0xBF). При чтении файла esbuild интерпретирует BOM как часть кода → ошибка синтаксиса.
- **Решение:** Удалён повреждённый файл, создан заново через инструмент `create_file` (UTF-8 без BOM).
- **Результат:** ✅ Файл корректно загружается, компонент рендерится.

## Итерация 13: Исправление "ничего не происходит" при нажатии кнопки

- **Проблема:** Кнопка "Тестировать все модели" не запускала тест. В консоли — тишина.
- **Причина 1:** `setIsLoading(true)` вызывался после `loadModels()`, а не перед. React batch-обновление откладывало рендер.
- **Причина 2:** FastAPI не принимал `model_id` как query-параметр — нужен явный `Query()`.
- **Решение:** Перенос `setIsLoading(true)` в начало; добавление `try/finally`; добавление `from fastapi import Query` и `model_id: str = Query(...)` в роутер.
- **Результат:** ✅ Кнопка работает, тесты запускаются.

## Итерация 14: Задержка между запросами и 3-статусная система

- **Цель:** Решить проблему массовых 429 (Rate Limit) при последовательном тестировании.
- **Проблема:** При быстром тестировании (без пауз) 70% моделей возвращали 429 — квота исчерпана.
- **Причина:** Free Tier Google AI API имеет строгие лимиты: от 5 RPM (Pro) до 30 RPM (Lite). Массовый прогон 36 моделей за 30 секунд исчерпывает все лимиты.
- **Решение:**
  1. Добавлен настраиваемый слайдер задержки (0–10 секунд, по умолчанию 3с)
  2. 429 выделен как отдельный статус ⏳ «квота исчерпана» (жёлтый цвет) — не красная ошибка
  3. Прогресс-бар показывает ETA с учётом задержки
  4. Сводка: 3 колонки (✅ Работает / ⏳ Квота / ❌ Ошибка)
- **Результат:** ✅ С задержкой 3с: 10 работают, 9 квота, 14 models.get, 1 мёртвая, 2 таймаут.

## Итерация 16: Тест 6 — Генерация видео (первая попытка)

- **Цель:** Проверить генерацию видео через 5 моделей Veo (2.0, 3.0, 3.0 Fast, 3.1, 3.1 Fast).
- **Методы:** 4 метода последовательно: generateContent → predictLongRunning → predict → generateVideos
- **Результат:** ❌ Все 4 метода неуспешны.
  - `generateContent` → 400: модальность `"VIDEO"` не поддерживается в AI Studio API
  - `predictLongRunning` → 400: `durationSeconds: 5` вне допустимого диапазона
  - `predict` → 404: Veo не поддерживает sync predict
  - `generateVideos` → пустой ответ (эндпоинт не существует)
- **Причина:** `durationSeconds` передавался как `5`, что не входит в `{4, 6, 8}` для Veo 3+.

## Итерация 18: Дискретные значения durationSeconds ← РЕШЕНИЕ

- **Открытие:** Из документации Google AI: `durationSeconds` принимает **ТОЛЬКО** дискретные значения:
  - Veo 2: `{5, 6, 8}`
  - Veo 3/3.1: `{4, 6, 8}`
- **Изменения:**
  1. `_get_valid_duration()` — маппит к ближайшему допустимому для модели
  2. UI: 3 кнопки (4с / 6с / 8с) вместо слайдера
  3. Default: 8с (универсально)
  4. Парсер: добавлен формат `generated_videos[0].video.uri`
- **Результат:** ✅ Все 5 моделей: `predictLongRunning` → 200. Операции создаются.
