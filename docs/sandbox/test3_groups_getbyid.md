# Тест 3: groups.getById — сравнение токенов

## Цель

Определить, какие данные возвращает метод `groups.getById` при использовании разных типов токенов: user token, community token, service key.

---

## Параметры запроса

| Параметр | Значение |
|---|---|
| `group_id` | `152651211` |
| `fields` | `members_count,description,status` |
| `v` | `5.199` |

---

## Тестируемая группа

| Поле | Значение |
|---|---|
| Название | Мега Вкусно - доставка суши роллов пицца Самара |
| screen_name | megavkusno_samara |
| Тип | page (публичная страница) |
| Подписчиков | 55 455 |

---

## Выводы

1. **Базовые данные** (название, описание, статус, количество подписчиков, аватар) — **идентичны** для всех 3 типов токенов.
2. **Персональные поля** (`is_admin`, `admin_level`, `is_member`, `is_advertiser`) — возвращает **только user token**. Эти поля показывают связь конкретного пользователя-владельца токена с группой.
3. **URL аватарок** — user token добавляет персонализированный параметр `&u=...` в URL фото. Community и service token возвращают общие URL.
4. **Метод доступен всем типам токенов** — ошибок нет ни у одного.
