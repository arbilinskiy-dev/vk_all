# Архитектура модуля Песочницы

Песочница — полностью **изолированный модуль** для тестирования VK API. Не использует основную бизнес-логику приложения (сервисы, CRUD, VK-клиент).

---

## Файловая структура

```
backend_python/
  routers/
    sandbox.py                          ← Роутер: /api/sandbox/*
  services/
    sandbox/
      photo_upload_test.py              ← Сервис: логика цепочки VK API

features/
  sandbox/
    components/
      SandboxPage.tsx                   ← Главная страница с вкладками
      tests/
        test1-photo-upload/
          PhotoUploadTest.tsx            ← UI компонент теста
          usePhotoUploadTest.ts          ← Хук: состояние + API вызовы
```

---

## Принцип работы

1. **Фронтенд** — форма с полями ввода (токены, group_id, файл). Отправляет `multipart/form-data` на бэкенд.
2. **Бэкенд** — последовательно выполняет цепочку VK API запросов, логируя каждый шаг (request, response, время, статус).
3. **Результат** — JSON со всеми шагами возвращается на фронтенд и отображается в виде раскрывающихся карточек `StepCard`.

---

## Навигация

Песочница доступна через боковую панель (`PrimarySidebar.tsx`). Состояние навигации: `AppView = 'sandbox'`. Роутер подключён в `main.py`.
