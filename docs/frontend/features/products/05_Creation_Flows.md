# Процессы создания товаров

В системе реализовано два сценария создания: для одного товара (детальный) и для множества (табличный).

## 1. Создание одного товара (`CreateSingleProductModal`)

**Хук:** `features/products/hooks/useCreateSingleProduct.ts`

### Особенности UI
*   **Фото:** Пользователь может загрузить файл с диска ИЛИ вставить прямую ссылку на изображение. Компонент `NewPhotoCell` валидирует ссылку (пытается загрузить картинку) перед сохранением.
*   **Описание:** Использует полноценный редактор (`DescriptionSection`) с поддержкой AI-генератора и переменных проекта.
*   **Настройки проекта:** Если пользователю нужно добавить переменную, он может открыть настройки проекта прямо из модального окна создания товара (поверх него).

### Логика сохранения
1.  Валидация обязательных полей (красная подсветка).
2.  Если выбрано фото-файл: оно загружается на сервер VK (`photos.getMarketUploadServer` -> save).
3.  Вызывается `market.add`.
4.  Если выбрана подборка, вызывается `market.addToAlbum`.
5.  Товар добавляется в локальный стейт (`setItems`), таблица обновляется без перезагрузки.

## 2. Массовое создание (`CreateMultipleProductsModal`)

**Хук:** `features/products/hooks/useCreateMultipleProducts.ts`

### Концепция "Динамической таблицы"
Интерфейс представляет собой таблицу, где каждая строка — это черновик нового товара (`NewProductRow`).
*   Изначально отображается 2 пустые строки.
*   Кнопка "+ Добавить товар" добавляет новую строку.
*   Кнопка "Копировать" (в строке) дублирует текущую строку (цену, категорию, подборку), но очищает название и фото.

### Валидация и Обработка ошибок
*   **Построчная валидация:** Перед отправкой проверяется каждая строка. Если есть ошибки, они подсвечиваются, и отправка блокируется.
*   **Toast + автопрокрутка:** При нажатии «Создать» с ошибками показывается toast с количеством проблемных строк. Первая строка с ошибкой прокручивается в видимую область через `scrollIntoView({ behavior: 'smooth', block: 'center' })`. Привязка по `data-tempid` на `<tr>` в `ProductRow.tsx`.
*   **Частичный успех:** Отправка происходит последовательно.
    *   Если 5 товаров сохранились, а 2 упали с ошибкой:
        *   Успешные товары удаляются из таблицы и появляются в основном списке.
        *   Неудачные товары остаются в таблице.
        *   Появляется консоль ошибок с детальным описанием причины для каждой строки.

### Прогресс-бар (release2)
Во время массового создания в фиксированном `<footer>` модального окна отображается:
*   Индigo-полоса прогресса с анимацией (`current / total`).
*   Счётчик: зелёная галочка для успешных, красный крестик для ошибок.
*   Название текущего товара с `truncate`.
*   Кнопка в footer показывает `X/Y` во время выполнения.

**Интерфейс:** `BulkCreationProgress { current, total, currentName, succeeded, failed, status }`.

### Отмена создания (AbortController) (release2)
Реализована через паттерн `AbortController`:
*   `abortControllerRef` в `useCreateMultipleProducts.ts`.
*   `handleCreateMultipleProducts` в `useProductsManager.ts` принимает `signal?: AbortSignal`, проверяет `signal.aborted` перед каждой итерацией.
*   Кнопка «Отмена» → «Остановить» во время создания.
*   При нажатии «Остановить» — `ConfirmationModal` с информацией: «Уже создано X из Y товаров».
*   Уже созданные товары сохраняются, оставшиеся помечаются «Отменено пользователем».

### Технические детали
*   Использует временные ID (`tempId = uuidv4()`) для отслеживания строк React-ом.
*   Фотографии хранятся в памяти (`File` объект) до момента сохранения.
*   После успешного создания вызывается `dataActions.handleRefreshAll()` для синхронизации с VK API.

## 3. Импорт из буфера обмена (`PasteFromClipboardModal`) (release2)

### TSV-парсер (`tsvParser.ts`)
Посимвольный конечный автомат по RFC 4180:
*   Ячейки в кавычках с переносами строк.
*   Экранирование кавычек (`""` → `"`).
*   Обрезка пробелов для каждой ячейки.

Заменяет наивный `text.split(/\r\n|\n|\r/)`, который ломал ячейки с переносами строк.

### Авто-маппинг колонок
Первая строка анализируется через `HEADER_MAP` (из `fileParser.ts`):
*   `название` → `title`, `цена` → `price`, `описание` → `description` и т.д.
*   Гарантируется уникальность: каждое поле маппится не более одного раза.
*   Занятые поля в дропдауне отображаются с пометкой «занято» и `disabled`.

### Кастомный dropdown (`ColumnMappingDropdown`)
Нативный `<select>` заменён на портал-компонент (`createPortal`):
*   Голубая indigo-обводка (`focus:ring-2 focus:ring-indigo-500`).
*   Визуальная галочка у выбранного пункта.
*   Закрытие по клику-вне/скроллу/ресайзу.

### Категория из экспорта
Двухступенчатая логика поиска:
1. Из значения извлекается ID в скобках (regex `/\((\d+)\)$/`).
2. Поиск по `c.id === extractedId`, fallback по `c.name`.

### Подборки
При импорте собираются уникальные названия подборок, проверяется их наличие в VK, отсутствующие создаются через `market/createAlbum`. Обновление через callback `onAlbumsCreated`.

### Фото URL
Добавлена опция «Фото (URL)» в маппинг колонок для обоих модальных окон.

## 4. Импорт из файла (`FileImportMappingModal`) (release2)

### CSV-парсер с кириллицей (`fileParser.ts`)
Функция `isBinaryExcelFile()` определяет формат по magic bytes:
*   **XLSX** = ZIP (`PK\x03\x04`), **XLS** = OLE2 (`0xD0CF11E0`) → `readAsArrayBuffer` + XLSX.
*   **CSV/TSV/текст** → `readAsText(file, 'UTF-8')`:
    *   Удаление BOM (`\uFEFF`).
    *   Автоопределение разделителя (`detectCSVDelimiter`: запятая, точка с запятой, табуляция).
    *   Полноценный CSV-парсер (`parseCSVText`) по RFC 4180.

### Фильтрация полей по режиму
Проп `mode?: 'create' | 'update'`:
*   `create` — скрываются поля `vk_id` и `vk_link` (бессмысленны для новых товаров).
*   `update` — все поля доступны.
*   Авто-маппинг также пропускает скрытые поля.

### Префикс «NEW»
Стейт `addNewPrefix` в `ProductsTab.tsx` запоминает выбор из `UploadOptionsModal`. При `addNewPrefix === true` каждому товару добавляется `NEW ` к `title`.

## 5. Фотографии товаров (release2)

### Skeleton + fade-in (`PhotoCell.tsx`)
*   Стейт `isLoaded` отслеживает загрузку каждого изображения.
*   Пока загружается — пульсирующий серый скелетон (`bg-gray-200 animate-pulse`).
*   По `onLoad` — плавное проявление (`opacity-0` → `opacity-100`, `transition-opacity duration-300`).

### VK CDN retry (бэкенд)
Функция `_download_photo_with_retry()` в `market_item_create.py`:
*   3 попытки, задержка 2с между ними для 500/502/503/504.
*   При исчерпании retry — `_build_smaller_photo_url()` уменьшает `cs=` (напр. `2560x0` → `1280x0`).
*   Signed URL (`/impg/` с `sign=`) не модифицируются (403 при изменении параметров).

### Выбор лучшего URL фото (бэкенд)
CRUD `market_crud.py` при синхронизации извлекает лучший URL из `photos[].sizes[]` с приоритетом: `w > z > y > x > r > q > p > o`. Fallback на `thumb_photo`.
