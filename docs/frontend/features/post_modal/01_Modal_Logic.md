# Модальное окно поста: Логика и Состояния

**Компонент:** `features/posts/components/modals/PostDetailsModal.tsx`
**Хук:** `features/posts/hooks/usePostDetails.ts`
**Хук формы:** `features/posts/hooks/usePostForm.ts`

## 1. SlidePanel вместо центрированного модального окна (release3)

Модальное окно поста переведено на компонент `SlidePanel` — выезжающую панель справа:
- **Компонент:** `shared/components/modals/SlidePanel.tsx`
- Анимация: панель выезжает справа налево с `cubic-bezier(0.16, 1, 0.3, 1)`.
- **Хлястик закрытия (close flap):** Кнопка-крестик прикреплена к левому краю панели вне самой панели.
- Скруглённые углы: `rounded-l-2xl`.
- Overlay: затемнённый полупрозрачный фон, закрытие по клику.
- Ширина: `w-full` на мобильных, `sm:w-[85vw]`, `md:w-[85vw]`.
- CSS-анимации: `slide-panel-in`, `slide-panel-out`, `slide-panel-overlay-in` (определены в `index.html`).

## 2. 4-колоночная раскладка (release3)

> **Изменение:** Ранее использовалась 3-колоночная раскладка. Теперь 4 колонки с заголовками `text-xs font-semibold text-gray-500 uppercase tracking-wider`.

| Колонка | Ширина | Содержимое |
|---------|--------|-----------|
| **НАСТРОЙКИ** | 30% | Табы публикации, закрепление на стене, PostCreationOptions, MultiProjectSelector, PostDateTimePicker |
| **AI-ПОМОЩНИК** | 25% | AIGenerator с `fillParent={true}`, AI-пресеты, мульти-генерация |
| **КОНТЕНТ** | 25% | PostTextSection (текст + переменные + эмодзи), AttachmentsDisplay (всегда видим), PollSection, PostMediaSection (фото/документы) |
| **ПРЕДПРОСМОТР** | 20% | PostPreview (карточка VK-стиль), сводка расписания |

Каждая колонка: `overflow-y-auto`, независимая прокрутка, разделение `border-r`.

## 3. Режимы работы (`mode`)

Модальное окно может находиться в одном из трех состояний:

1.  **`view` (Просмотр):** Все поля `readOnly`, кнопки сохранения скрыты. Кнопка "Редактировать" → `edit`.
2.  **`edit` (Редактирование):** Поля доступны для изменения. Активна логика "Dirty Check".
3.  **`copy` (Копирование):** Данные из исходного поста, новый ID (`new-post-...`). Дата сбрасывается на "Завтра 10:00", если исходная в прошлом.

## 4. "Грязное" состояние (`useDirtyCheck`)

*   **Сравниваются:** Текст, список ID изображений (порядок не важен), вложения, дата/время, проекты (в мульти-режиме), `firstCommentText` (release3).
*   **Влияние:** Если `isDirty === false`, кнопка "Сохранить" неактивна (кроме создания).
*   **Защита:** При `isDirty === true` → `ConfirmUnsavedChangesModal`.

## 5. Безопасное закрытие

`handleClose` проверяет `isDirty` и состояние UI (AI-панель, переменные). Предупреждение перед закрытием, чтобы не потерять контекст.

## 6. Блокировка UI
Во время сохранения (`isSaving`) или загрузки медиа (`isUploading`): кнопки и поля блокируются.

## 7. Новые элементы интерфейса (release3)

### Табы способа публикации
3 кнопки в контейнере `bg-gray-200 rounded-lg`:
- **Запланировать** — стандартный таймслот
- **В отложку VK** — черновик в отложенные записи
- **Опубликовать сейчас** — немедленная публикация

### Тумблер «Закрепить на стене»
Переключатель с иконкой 📌. При включении — бейдж «Закреплён» в заголовке «Предпросмотр».

### Первый комментарий от имени сообщества
- Тумблер в колонке настроек (видим при наличии `communityToken`).
- `CommentTextEditor` в колонке контента — полноценный редактор: emoji-пикер, undo/redo, вставка переменных, счётчик (4096).
- Превью комментария в `PostPreview`: аватар + название + бейдж «Автор» + текст.

### Переменные — отображаются всегда
Кнопка-toggle удалена. Секция переменных видна постоянно. Автозагрузка при монтировании.

> **Цвета кнопки «+ Добавить»:** `border-indigo-400 text-indigo-600 hover:bg-indigo-50` (исправлено с `blue-*` на `indigo-*` по дизайн-системе).

### Реорганизация колонки «Контент» (release4)

> **Изменение:** Порядок элементов в колонке «Контент» перестроен для лучшего UX.

Новый порядок сверху вниз:
1. **PostTextSection** — текст поста, переменные, эмодзи
2. **AttachmentsDisplay** — карточки вложений (документы, видео, опрос). Вынесен из PostMediaSection и всегда видим под текстом.
3. **PollSection** — форма создания/редактирования опроса. Размещён рядом с карточкой опроса для логической группировки.
4. **PostMediaSection** — загрузка фото/видео (`showAttachments={false}`, т.к. AttachmentsDisplay уже отрисован выше)
5. **CommentTextEditor** — первый комментарий

Prop `showAttachments?: boolean` (default `true`) у PostMediaSection позволяет скрыть встроенный AttachmentsDisplay при его размещении отдельно.

### PostPreview (VK-style предпросмотр)
Карточка поста в стиле ВКонтакте:
- Шапка: реальная аватарка сообщества (`avatar_url`) + название (`vkGroupName`) + дата.
- Текст: `whitespace-pre-wrap`, подстановка глобальных переменных, VK-разметка через `renderVkFormattedText`.
- Сетка изображений: 1 фото → `aspect-square`, 2 → `grid-cols-2`, 3 → `grid-cols-3`, 4+ → `grid-cols-2` с оверлеем `+N`.
- Футер VK-стайл: декоративные кнопки лайк/комментарий/шер.
- Пустое состояние: иконка глаза + подсказка.

### Undo/Redo текста (`useTextUndoHistory`)
- Хук: `shared/hooks/useTextUndoHistory.ts` (100 снапшотов, дебаунс 300мс).
- Ctrl+Z / Ctrl+Shift+Z / Ctrl+Y.
- Кнопки «Отменить» / «Повторить» со стрелками рядом с лейблом «Текст публикации».

### Inline эмодзи-пикер
`EmojiPicker` проп `inline=true` — растягивается внутри контейнера без border/shadow. Рендерится между тулбаром и textarea.

### Текстовые кнопки AI-помощника
SVG иконки заменены на: «Regenerate» (gray), «Копировать» (blue), «Добавить» (green), **«Заменить»** (indigo).

### Убраны кнопки B/I/U/S
VK не поддерживает markdown/BB-code. Кнопки удалены. Счётчик символов (N/8206) перемещён в тулбар справа.
