## 1. Общие сведения

### 1.1. Назначение системы

**VK Content Planner** — это веб-платформа (SaaS) для централизованного управления контентом, коммуникациями и автоматизацией бизнес-процессов в сообществах социальной сети ВКонтакте.

Система предназначена для SMM-специалистов, контент-менеджеров, аккаунт-менеджеров и руководителей отделов маркетинга, работающих одновременно с множеством сообществ ВК (до 200+ проектов).

### 1.2. Бизнес-цели

| # | Цель | Описание |
|---|---|---|
| 1 | **Централизация** | Единая точка управления всеми сообществами ВК вместо разрозненных инструментов и ручных действий в интерфейсе ВК |
| 2 | **Экономия времени** | Массовые операции, шаблоны, автоматизации, AI-генерация — сокращение рутинных задач на 60-80% |
| 3 | **Контроль качества** | Единый стандарт контента через AI-коррекцию, глобальные переменные, контексты проектов |
| 4 | **Прозрачность работы** | Статистика действий менеджеров, KPI, логи — полная видимость кто/что/когда сделал |
| 5 | **Автоматизация коммуникаций** | Управление сообщениями всех сообществ из единого интерфейса, шаблоны ответов, real-time уведомления |
| 6 | **Маркетинговые активности** | Конкурсы, розыгрыши, рассылка промокодов — без ручной работы |
| 7 | **Управление товарами** | Работа с VK Market: массовое редактирование, импорт/экспорт, AI-категоризация |

### 1.3. Целевая аудитория

- **SMM-менеджер** — планирует и публикует контент, работает с расписанием
- **Аккаунт-менеджер** — отвечает на сообщения пользователей, работает с CRM-частью
- **Руководитель / Администратор** — контролирует метрики, управляет доступами, настраивает систему
- **Маркетолог** — запускает конкурсы, работает с товарами, анализирует статистику

### 1.4. Ключевые бизнес-метрики

- Количество управляемых проектов (сообществ ВК)
- Количество публикуемых постов в месяц
- Среднее время ответа на сообщение
- Количество обработанных диалогов в день
- Процент автоматизированных операций

## 2. Архитектура системы (общие требования)

### 2.1. Тип системы

Клиент-серверное веб-приложение (SPA — Single Page Application):
- **Фронтенд** — интерактивный клиент в браузере
- **Бэкенд** — REST API сервер
- **База данных** — реляционная СУБД
- **Внешние интеграции** — VK API, Google Gemini AI, SMTP

### 2.2. Технологический стек

| Слой | Технология | Обоснование |
|---|---|---|
| Фронтенд | React + TypeScript + Tailwind CSS | Быстрая разработка, типизация, консистентный UI |
| Сборщик | Vite | Скорость горячей перезагрузки, ESM |
| Бэкенд | Python + FastAPI | Асинхронность, авто-документация API, Pydantic |
| ORM | SQLAlchemy | Мощный ORM, поддержка миграций, кастомные типы |
| БД (разработка) | SQLite | Нулевая настройка, файловая БД |
| БД (продакшен) | PostgreSQL (Yandex Managed) | Надёжность, масштабируемость, managed-сервис |
| AI-движок | Google Gemini API | Генерация/коррекция текстов, подбор категорий |
| Кэширование / Pub-Sub | Redis (опциональный) | SSE-синхронизация между инстансами, лок-менеджмент |
| Шифрование | Fernet (AES-128 CBC) | Шифрование чувствительных данных в БД |
| Деплой | Yandex Cloud (Container + S3) | Российский провайдер, соответствие ФЗ-152 |

### 2.3. Окружения

| Окружение | Назначение | Инфраструктура |
|---|---|---|
| Локальное | Разработка и отладка | SQLite, localhost:8000 / localhost:5173 |
| Предпродакшен | Тестирование перед выкаткой | VM в Yandex Cloud, PostgreSQL |
| Продакшен | Боевая система | Serverless Container, Managed PostgreSQL, S3 |

### 2.4. Принципы архитектуры

1. **Feature-Sliced Design** — код организован по бизнес-фичам, а не по типу файлов
2. **Трёхслойная архитектура бэкенда**: Router → Service → CRUD / External API
3. **Facade (Hub) паттерн** — тонкие файлы-хабы реэкспортируют логику из подмодулей
4. **Оптимистичный UI** — данные из кэша показываются мгновенно, обновления в фоне
5. **Шифрование At Rest** — токены и ключи шифруются в БД, не хранятся в открытом виде
6. **Auto-миграции** — схема БД обновляется автоматически при старте приложения
7. **Авто-документация** — Swagger UI генерируется из Pydantic-схем

## 3. Модули системы

### 3.1. Аутентификация и управление доступом

**Назначение:** контроль доступа к системе, разграничение прав.

**Функциональность:**
- Вход по логину/паролю (сессионная авторизация, cookie)
- Вход через VK ID (OAuth 2.0 + PKCE)
- Ролевая модель: `Администратор` и `Пользователь`
- Управление пользователями (CRUD) — создание логинов, транслитерация ФИО
- Привязка VK-аккаунтов к учётным записям системы
- Логи авторизации (когда, откуда, кто входил)
- Управление активными сессиями (просмотр, принудительное завершение)
- Дашборд активности пользователей
- Аналитика использования AI-токенов по пользователям

---

### 3.2. Проекты (сообщества ВК)

**Назначение:** центральная сущность системы — каждый «проект» это одно сообщество ВКонтакте.

**Функциональность:**
- Боковая панель со списком проектов и счётчиками (запланировано, предложено, непрочитанные сообщения)
- Настройки проекта: VK-токены сообщества, ссылки, команда, интеграции
- Массовое добавление проектов по URL-адресам групп ВК
- Архивирование / восстановление / полное удаление проектов
- Контекст проекта — структурированная информация о бизнесе (для AI): описание компании, продукты, тон голоса, целевая аудитория
- AI-автозаполнение контекста на основе контента сообщества
- Глобальные переменные — макросы-подстановки (`{{имя_магазина}}`, `{{телефон}}`) для шаблонизации постов
- Администрируемые группы VK — автоопределение списка доступных сообществ

---

### 3.3. Расписание контента (Календарь)

**Назначение:** визуальное планирование публикаций в формате календарной сетки.

**Функциональность:**
- Календарная сетка по дням (7-колоночный вид, навигация по неделям)
- Отображение постов по статусам: запланированные, опубликованные, системные
- Drag-and-drop перемещение постов между датами
- Создание / редактирование / удаление постов из календаря
- Заметки к датам (привязка к конкретным дням)
- Публикация постов: мгновенная или отложенная
- Закрепление / открепление постов
- Навигация к конкурсам, AI-постам, товарам из расписания
- Индикатор обновлений при изменениях из VK (Callback API)

---

### 3.4. Посты (Контент)

**Назначение:** полноценный редактор публикаций с поддержкой медиа, AI и массовых операций.

**Функциональность:**
- Создание / редактирование постов с текстом, медиа, геолокацией
- Загрузка фото и видео (3-шаговая загрузка через VK API)
- AI-генерация текста поста (Google Gemini) с учётом контекста проекта
- AI-коррекция и обработка текста
- Пресеты AI-промптов (сохраняемые настройки генерации)
- Опросы (создание через VK API)
- Мультипроектная публикация — одновременная отправка поста в несколько сообществ
- Массовое редактирование постов (поиск-замена по текстам, Bulk Edit)
- Система тегов для категоризации постов
- Предложенные посты — просмотр и модерация предложки сообщества
- Массовая AI-коррекция предложенных постов

---

### 3.5. Системные посты

**Назначение:** автоматическая публикация постов по расписанию без участия VK (внутренняя очередь).

**Функциональность:**
- Создание постов с датой/временем публикации
- Фоновый процесс публикации (проверка каждые ~50 секунд)
- Поддержка циклических постов (повторяющиеся публикации)
- Первый комментарий — автоматический комментарий от имени сообщества после публикации
- Подтверждение / отмена / ручная публикация системного поста

---

### 3.6. Товары (VK Market)

**Назначение:** полноценное управление каталогом товаров VK-маркета из единого интерфейса.

**Функциональность:**
- Таблица товаров с настраиваемыми колонками, сортировкой, поиском
- Фильтрация по альбомам (категориям)
- Создание / редактирование / удаление товаров
- Массовое редактирование: цены, описания, фото, статусы
- Импорт товаров из файлов (CSV, XLSX) с маппингом полей
- Экспорт каталога
- Обновление товаров из файла с сопоставлением
- Управление альбомами (создание, редактирование, удаление)
- AI-подбор категорий товаров (двухэтапный: секция → подкатегория)
- AI-массовая коррекция заголовков и описаний товаров
- Загрузка фото товаров через VK API

---

### 3.7. Сообщения (CRM / Аккаунт-менеджмент)

**Назначение:** модуль управления коммуникациями с пользователями ВК — полноценная CRM-система для обработки обращений.

**Функциональность:**

**Чат:**
- Список диалогов с фильтрами (все / непрочитанные / важные)
- Real-time обмен сообщениями через SSE (Server-Sent Events)
- Индикатор набора (typing) в реальном времени
- Фокус менеджеров — видно, кто из сотрудников сейчас в каком диалоге
- Отправка текстовых сообщений и вложений
- Пометка диалогов как «Важное»
- Пометка прочитано / непрочитано

**Метки (ярлыки) диалогов:**
- CRUD меток с цветовой кодировкой
- Назначение / снятие меток с диалогов
- Фильтрация диалогов по меткам

**Шаблоны быстрых ответов:**
- Создание / редактирование / удаление шаблонов
- Вставка шаблона в поле ввода
- Превью шаблона

**Промокоды:**
- Управление промо-листами (группы промокодов)
- Добавление / удаление отдельных кодов
- Интеграция с шаблонами сообщений

**Статистика сообщений:**
- Кросс-проектный дашборд: входящие / исходящие, подписки / отписки
- Графики по периодам (часы → дни, авто-гранулярность)
- Статус сообщений по проектам
- Статистика по конкретным пользователям

**Мониторинг действий сотрудников (АМ-анализ):**
- KPI менеджеров: количество ответов, среднее время реакции
- Круговые диаграммы распределения нагрузки
- Графики активности по дням
- Детализация по каждому менеджеру

### 3.8. Автоматизации

**Назначение:** модуль автоматических сценариев для маркетинговых активностей.

#### 3.8.1. Автоматизация историй (Stories)
- Автопубликация VK-историй из постов по ключевым словам
- Ручная публикация историй
- Статистика просмотров историй (reach, viewers)
- Настройка параметров автоматизации
- Логи автоматизации

#### 3.8.2. Конкурс отзывов
- Автоматический конкурс по хэштегу `#отзыв`
- Автонумерация участников
- Автоматический розыгрыш победителей
- Массовая рассылка промокодов победителям через личные сообщения
- Чёрный список (исключение пользователей из конкурсов)
- Логи доставки промокодов

#### 3.8.3. Универсальные конкурсы
- Создание произвольных конкурсов с редактором условий
- Сбор участников по критериям (лайки, комментарии, репосты)
- Выбор победителей (случайный / ручной)
- Промокоды и рассылка

#### 3.8.4. Конкурсы 2.0
- Обновлённая версия с расширенным редактором
- Deep-linking для генерации ссылок участия

#### 3.8.5. AI-посты
- Генерация постов через AI (Google Gemini)
- Список и управление сгенерированными постами
- Автоматическая публикация по расписанию

## 5. Безопасность

### 5.1. Требования

| # | Требование | Реализация |
|---|---|---|
| 1 | Контроль доступа | Сессионная авторизация (cookie), ролевая модель (admin/user) |
| 2 | Шифрование данных | Fernet (AES-128 CBC) для токенов и ключей в БД |
| 3 | Ротация ключей | Поддержка двух ключей шифрования с автомиграцией |
| 4 | CORS-политика | Whitelist конкретных доменов (prod, preprod, local) |
| 5 | Маскировка секретов | Токены отображаются в UI в маскированном виде |

## 10. Глоссарий

| Термин | Определение |
|---|---|
| **Проект** | Одно сообщество (группа) ВКонтакте, подключённое к системе |
| **Системный пост** | Пост, управляемый внутренней очередью системы (а не через VK API postpone) |
| **Глобальная переменная** | Макрос-подстановка (`{{переменная}}`), единожды определённый и доступный во всех постах проекта |
| **Контекст проекта** | Структурированная информация о бизнесе (описание, продукты, tone of voice) для AI-генерации |
| **Callback API** | Механизм VK для real-time уведомлений о событиях в сообществе |
| **SSE** | Server-Sent Events — протокол push-уведомлений от сервера к клиенту |
| **Пресет промпта** | Сохранённая конфигурация AI-генерации (системный промпт, температура, модель) |
| **Промо-лист** | Группа промокодов, привязанная к проекту, для рассылки в конкурсах и шаблонах |
| **АМ-анализ** | Мониторинг действий аккаунт-менеджеров: KPI, время ответа, объём обработки |
| **Bulk Edit** | Массовое редактирование — поиск-замена по текстам постов с параллельными воркерами |
| **Fernet** | Симметричный алгоритм шифрования (AES-128 CBC + HMAC), используемый для защиты токенов |
